## ZoteroScript-P4-LR4 文献综述 V1

```Java
#📚LR[color=#0EA293][trigger=]

These are Zotero item informations:

${
(async function() {
  let items = ZoteroPane.getSelectedItems()
  const docs = Meet.Zotero.items2documents(items)
  
  // 为每个文档添加标签信息
  const docsWithTags = docs.map((doc, idx) => {
    const item = items[idx]
    const tags = item.getTags().map(tag => tag.tag).join(', ')
    
    // 将标签信息添加到 pageContent 中
    return {
      …doc,
      pageContent: doc.pageContent + (tags ? `\nTags: ${tags}` : '\nTags: No tags')
    }
  })
  
  Meet.Global.views.insertAuxiliary(docsWithTags)
  
  return docsWithTags.map(
    (doc, index) => "[" + String(index + 1) + "]" + " " +  doc.pageContent
  ).join("\n\n")
})()
}$

## 各列填充指南

### 第1列：主要研究议程
- **定义**：表格的"行"，代表高阶的、逻辑平行的研究流派或子领域
- **数量**：3-5 个
- **要求**：各议程之间逻辑平行、互不重叠
- **示例**（假设主题为"AI与创新"）：
  - 议程1：AI作为创新驱动力
  - 议程2：AI作为创新产物  
  - 议程3：AI采纳与组织能力
  - 议程4：AI重塑创新范式

### 第2列：核心研究问题
- **定义**：该议程试图回答的最核心理论问题
- **格式**：
  - 如果只有一个核心问题，用一句简洁的疑问句或陈述句
  - 如果有多个核心问题，使用序号 (1) (2) (3) 分点列出
  - 每一点必须高度凝练
  - 如果问题之间存在逻辑关系，按照逻辑顺序排列
- **示例**：
  - 单一问题："AI如何影响企业创新绩效？"
  - 多个问题："(1) AI如何影响创新投入？ (2) AI如何影响创新产出？"

### 第3列：研究设计维度
- **定义**：该列整合了研究设计的四个关键维度，用分行展示，使表格布局更加美观紧凑
- **格式要求**：必须包含以下四个子维度，使用 `<br>` 标签换行，每个子维度加粗标题：
  
  1. **[主题]角色**：
     - 明确指出 `${Meet.Global.input}$` 在该议程中扮演的**变量角色**
     - 使用标准术语：自变量 (IV) / 因变量 (DV) / 中介变量 (Mediator) / 调节变量 (Moderator) / 背景/情境 (Context)
     - ⚠️ **这是确保各议程逻辑平行的关键维度**
  
  2. **研究方法**：
     - 该议程学者最常使用的研究方法
     - 简洁列举，用分号分隔
  
  3. **分析层次**：
     - 该议程研究聚焦的分析层次
     - 选项：微观 / 中观 / 宏观 / 多层次
  
  4. **核心理论**：
     - 该议程最常依赖的核心理论或关键概念
     - 简洁列举，用分号分隔

- **示例**：
  ```

  **[主题] 角色**：自变量 (IV)<br>**研究方法**：面板数据回归；多案例研究<br>**分析层次**：微观（企业层）<br>**核心理论**：资源基础观；动态能力理论

  ```

### 第4列：主要研究内容与发现
- **定义**：该议程下的主要研究内容和关键发现的综合归纳
- **格式要求**：
  - 使用序号 (1) (2) (3) 分点列出
  - 不要简单罗列单篇论文，而要**提炼共性规律和争论焦点**
  - **每一点必须高度凝练**，去除冗余表述，直击核心内容
  - **如果各点之间存在逻辑关系**（如因果关系、递进关系、对比关系等），则按照逻辑顺序排列
  - 每个要点后必须标注支持文献，格式：**(作者, 年份)[编号]**
- **示例**：
  ```

  (1) AI 采纳提升组织创新能力 (Smith, 2020)[1]  
  (2) 效果受组织吸收能力调节 (Jones, 2021)[3]  
  (3) 存在技术 - 组织匹配挑战 (Lee, 2022)[5]

