(async function() {
  let items = ZoteroPane.getSelectedItems()
  const docs = Meet.Zotero.items2documents(items)
  
  // 为每个文档添加标签信息
  const docsWithTags = docs.map((doc, idx) => {
    const item = items[idx]
    const tags = item.getTags().map(tag => tag.tag).join(', ')
    
    // 将标签信息添加到 pageContent 中
    return {
      ...doc,
      pageContent: doc.pageContent + (tags ? `\nTags: ${tags}` : '\nTags: No tags')
    }
  })
  
  Meet.Global.views.insertAuxiliary(docsWithTags)
  
  return docsWithTags.map(
    (doc, index) => "[" + String(index + 1) + "]" + " " +  doc.pageContent
  ).join("\n\n")
})()
}$

## 1. 核心原则

**穷尽式原则 (Exhaustive Principle)**：这是最高指导原则。你扮演的是一个**穷尽式的信息提取器**，而非选择性的评论者。你的核心任务是**遍历用户提供的所有文献**。对于在文献中识别出的**每一个不同的观点或发现**，你必须**罗列出所有支持该观点的文献**，绝不可以自作主张地进行筛选或仅选取部分文献。

**星级优先原则 (Star Priority Principle)**：这是一个**纯粹的排序和呈现规则**，绝不能与"穷尽式原则"冲突。具体做法是：
- 在"**文献综述段落**"列中，当一个观点有多篇文献支持时，你必须罗列出**所有**这些文献，并按照以下顺序排列：
  1. **首先按标签中的⭐️数量排序**：星星数量越多，优先级越高（5星＞4星＞3星＞2星＞1星＞无星标）
  2. **相同星级按年份排序**：在同一星级内，按照**年份从新到旧**的顺序排列

**学术精炼原则 (Academic Conciseness Principle)**：这是一个贯穿始终的风格要求。在确保信息完整（遵循穷尽式原则）的前提下，你的语言必须**高度精炼、专业且客观**。
- 特别是在"**文献综述段落**"列中，应避免冗余的表达和不必要的修饰，使用最直接的学术语言来呈现信息，以适应表格的紧凑格式。

## 3. 任务要求

请你围绕【用户输入的研究主题】，对【用户提供的文献集合】进行一次严格、客观、穷尽式的文献内容提取，并以一个包含**两列**的"文献综述矩阵"表格形式呈现你的分析结果。

### 3.1 分析维度（11个）

1. **研究背景** (如政策变革、技术范式转移、宏观经济冲击、社会文化趋势、全球化与地缘政治等)
2. **构念定义** (如核心定义、维度划分、内涵演变、竞争性构念、新兴构念等)
3. **测量方法** (如量化指标(熵指数/赫芬达尔指数)、档案数据(专利/财务)、调查问卷(李克特量表)、定性编码(案例/访谈)、文本挖掘等)
4. **理论视角** (如资源基础观(RBV)、动态能力理论、组织学习理论、制度理论、网络理论、交易成本经济学等)
5. **对象层次** (如国家、区域/产业集群、产业、企业、业务单元/项目、团队、个人等)
6. **研究情境** (如发达经济体 vs. 新兴市场、高科技产业 vs. 传统产业、制造业 vs. 服务业、企业所有制(国有/民营/家族)、特定历史时期(危机前后)等)
7. **研究方法** (如定量研究(面板回归/结构方程模型/生存分析)、定性研究(案例比较/扎根理论)、混合方法、实验设计、模拟仿真等)
8. **前因因素** (总结研究主题受到哪些因素影响) (如外部环境因素(环境动态性/竞争强度)、组织内部资源与战略(组织冗余/战略导向)、高管团队特征(TMT)、网络关系(联盟/网络位置)等)
9. **作用后果** (总结研究主题对哪些因素产生影响) (如财务绩效、创新绩效(探索式/利用式)、市场绩效、组织生存、组织韧性、组织学习与适应性等)
10. **中介机制** (识别并总结解释"如何"起作用的内在机制) (如吸收能力、知识整合、组织学习、动态能力、探索与利用平衡等)
11. **调节因素** (识别并总结影响效果强弱的边界条件) (如环境不确定性、制度环境、组织冗余、组织文化、企业规模/年龄、网络位置等)

### 3.2 输出要求

**文献综述段落 (Literature Review Paragraph)**：将**穷尽式罗列出的完整发现**，整合成一段逻辑清晰、语言精炼、能反映研究全貌（包括共识、争议和所有观点）的学术段落。**对于每一个发现，都必须穷尽式地罗列出所有支持该观点的文献引用**，并遵循"最新优先"原则进行排序。格式为：`（作者，年份，期刊缩写)编号`。

#### 3.2.1 段落结构要求
1. **开场陈述**：以该维度的核心主题或研究焦点开头，简洁概述该维度在研究领域中的重要性或地位
2. **主要发现**：系统性地呈现该维度的核心发现，包括：
   - **共识性观点**：明确标注为"核心共识"、"广泛认同"等，并列出所有支持文献
   - **争议性观点**：明确标注为"存在争议"、"结论不一"等，并分别列出不同观点的支持文献
   - **新兴观点**：标注为"新近研究"、"最新发现"等，突出研究前沿
3. **学术脉络**：通过文献年份的引用顺序，自然呈现该维度研究的时间演进和理论发展轨迹
4. **总结过渡**：以该维度对整体研究主题的意义或与其他维度的关联作结

#### 3.2.2 语言风格要求
1. **客观性**：使用第三人称，避免主观判断，以"研究发现"、"文献表明"等表述
2. **精炼性**：每句话都承载实质信息，避免冗余表达
3. **逻辑性**：使用适当的连接词（然而、此外、因此等）确保段落流畅
4. **专业性**：使用准确的学术术语，保持一致的表达方式

#### 3.2.3 文献引用要求
1. **完整性**：每个观点都必须有完整的文献支撑，不得遗漏
2. **星级优先**：首先按⭐️数量排序（5星＞4星＞3星＞2星＞1星＞无星标）
3. **时效性**：在同一星级内，优先引用最新文献，但必须包含奠基性文献
4. **权威性**：在星级和年份相同的情况下，优先引用顶级期刊或该领域公认的重要文献
5. **格式统一**：严格按照"（作者，年份，期刊缩写)编号"格式，多个文献用分号分隔
6. **缺省期刊**：无期刊来源（如书籍、报告）或无法确定期刊时，可省略期刊缩写

请确保最终输出是一个完整的、填充了所有内容的Markdown表格。
