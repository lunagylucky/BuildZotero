## Table2- 统计标签 版本迭代说明

### 概述
Table2- 统计标签模块用于统计和导出 Zotero 中文献条目的标签信息，支持按标签前缀分组统计，生成 Markdown 表格。该模块支持多种输入方式（直接输入前缀、数字编码选择、自定义前缀）和多种输出格式（期刊+作者格式、文献编号格式）。表格可以按标签出现次数排序，并提供年份、期刊、作者或文献编号等元数据信息。

**注意：** F 系列（V5-FO, V5-FA）是在 V 系列基础上添加关键词筛选功能的版本，其中 FA 表示 AND 逻辑（同时包含所有关键词），FO 表示 OR 逻辑（包含任意一个关键词）。所有版本文件均位于本文件夹中。所有版本的标签开头统一为 `Ttags`。


### 版本号体系说明
统计标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major]` 和 `V[major]-F[variant]`
- **V[major]（标准版本）**：表示核心功能版本，决定输入方式（直接输入前缀、数字编码选择）、变量标签前缀（旧版 IV/DV/MV 或新版 A1-DV/A2-IV 等）、输出格式（期刊+作者格式、文献编号格式）、items2documents 集成、条目细节标签列数（4列或5列）等
  - V1：直接输入前缀，输出期刊+作者格式
  - V2：数字编码选择（旧版变量标签前缀），输出期刊+作者格式
  - V3：数字编码选择（新版变量标签前缀），集成 items2documents，输出文献编号格式，条目细节标签 4 列
  - V4：基于 V3，条目细节标签 4 列（简化版，与 V3 相同）
  - V5：基于 V4，条目细节标签从 4 列扩展到 5 列（新增 V5-sor）
  - V6：基于 V5，条目细节标签保持 5 列（与 V5 相同）

- **V[major]-F[variant]（筛选版本）**：表示在 V 系列基础上添加关键词筛选功能
  - F 表示 Filter（筛选）
  - FA 表示 AND 逻辑（同时包含所有关键词）
  - FO 表示 OR 逻辑（包含任意一个关键词）
  - **V5-FO**：基于 V5，添加关键词筛选（OR 逻辑），条目细节标签从 5 列回退到 4 列
  - **V5-FA**：基于 V5，添加关键词筛选（AND 逻辑），条目细节标签保持 5 列

### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-10-13）
**核心特性：**
- 基础的标签统计功能
- 用户直接输入标签前缀（支持多个前缀，逗号分隔）
- 输出格式：序号、标签内容、次数、年份、期刊、作者
- 按标签出现次数排序（从高到低）
- 按前缀分组展示

**输入方式：**
- 用户直接输入标签前缀，支持多个前缀用逗号分隔
- 示例：`Theory/, sMeth/, Sample/, Item/, Result/`

**输出格式：**

```markdown
## Theory/
| 序号 | 标签内容 | 次数 | 年份 | 期刊 | 作者 |
|------|----------|------|------|------|------|
| 1 | Dynamic Capability Theory | 4 | 2020, 2021, 2022 | Strategic Management Journal(2020,2021) | David J. Teece(2020); ... |
```

**标签格式：**
- 标签内容不包含前缀部分（去掉前缀）
- 年份排序并用逗号连接
- 期刊格式：`期刊名称(年份1,年份2)`
- 作者格式：`作者全名(年份1,年份2)`

**特点：**
- ✅ 基础功能完整
- ✅ 支持多个前缀分组
- ✅ 按出现次数排序
- ❌ 需要手动输入前缀（不够便捷）
- ❌ 没有集成 items2documents（文献不可点击）

- 使用旧版变量标签前缀（IV/, DV/, MV/, Mod/, Med/, CV/）

**输入方式：**
- 用户输入数字编码，支持多个编码用逗号分隔
- 示例：`1,3,4` 或 `1-1,1-2,3,5-1`

**输出格式：**
- 与 V1 相同：序号、标签内容、次数、年份、期刊、作者

**注意：** F 系列是在 V 系列基础上添加关键词筛选功能的版本：

#### V5-FO - 前缀关键词筛选（OR 逻辑）（2025-10-17）
**核心特性：**
- **基于 V5**，添加前缀关键词筛选功能
- 关键词筛选使用 OR 逻辑（包含任意一个关键词）
- 数字编码选择 + 自定义前缀（与 V5 相同）
- 集成 items2documents 和辅助视图

- 移除起始序号（normalizedContent）
- **条目细节标签 4 列**（从 V5 的 5 列回退到 4 列）

**关键词筛选功能：**

```javascript
// 用户输入选择 - 第一步：选择前缀
const userInput = prompt("【第 1 步】请输入要统计的标签类型...");

// 用户输入选择 - 第二步：输入关键词
const keywordInput = prompt("【第 2 步】请输入要筛选的关键词（可选）：\n\n" +
  "- 留空：显示所有匹配前缀的标签\n" +
  "- 单个关键词：如 平台\n" +
  "- 多个关键词：用逗号分隔，如 平台,企业,竞争\n" +
  "- 显示结果：包含任意一个关键词的标签\n\n" +
  "提示：关键词不区分大小写");

// 关键词筛选逻辑（OR 逻辑）
if (keywords.length > 0) {
  // 如果设置了关键词，检查标签内容是否包含任意一个关键词
  const tagContentLower = tagContent.toLowerCase();
  const hasKeyword = keywords.some(kw => tagContentLower.includes(kw));
  
  // 不包含任何关键词，跳过此标签
  if (!hasKeyword) {
    continue;
  }
}
```

**关键词筛选逻辑：**
- OR 逻辑：包含任意一个关键词的标签都会显示
- 示例：关键词"平台,企业,竞争"，标签包含"平台"或"企业"或"竞争"中的任意一个都会显示
- 关键词不区分大小写

**统计信息增强：**
- 统计信息中显示筛选条件（前缀、关键词）

**与 V5 的区别：**

| 特性 | V5 | V5-FO |
|------|----|----|
| 基础版本 | - | 基于 V5 |
| 关键词筛选 | ❌ 无 | ✅ 有（OR 逻辑） |
| 条目细节标签列数 | 5 列 | 4 列（回退） |
| 输入步骤 | 1 步（选择前缀） | 2 步（选择前缀 + 输入关键词） |
| 筛选灵活性 | ❌ 无 | ✅ 支持关键词筛选 |

**特点：**
- ✅ 基于 V5，添加前缀关键词筛选功能（OR 逻辑）
- ✅ 两步输入流程（先选择前缀，再输入关键词）

#### V5-FA - 前缀关键词筛选（AND 逻辑）（日期未知）
**核心特性：**
- **基于 V5**，添加前缀关键词筛选功能
- 关键词筛选使用 AND 逻辑（同时包含所有关键词）
- 数字编码选择 + 自定义前缀（与 V5 相同）
- 集成 items2documents 和辅助视图

**关键词筛选逻辑变化：**

```javascript
// V5-FO: OR 逻辑
const hasKeyword = keywords.some(kw => tagContentLower.includes(kw));

// V5-FA: AND 逻辑
const hasAllKeywords = keywords.every(kw => tagContentLower.includes(kw));
```

**关键词筛选逻辑：**
- AND 逻辑：必须同时包含所有关键词的标签才会显示
- 示例：关键词"平台,企业,竞争"，标签必须同时包含"平台"和"企业"和"竞争"才会显示
- 关键词不区分大小写

**统计信息增强：**
- 统计信息中明确标注"同时包含"（且关系）
- 未找到匹配标签时提供提示信息
