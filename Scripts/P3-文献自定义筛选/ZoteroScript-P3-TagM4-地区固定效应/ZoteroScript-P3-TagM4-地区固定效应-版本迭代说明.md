## TagM4-地区固定效应 版本迭代说明

### 概述
TagM4-地区固定效应模块用于根据条目的 `A2-IV/` 和 `A6-CV/` 标签内容，自动添加固定效应类型标签（省份、年份、行业）。该模块支持两种操作模式：仅添加标签（V1）和智能清理（V2，会删除不匹配的标签）。

**注意：** 所有版本文件均位于本文件夹中。所有版本的标签开头统一为 `FE`。


### 版本号体系说明
地区固定效应脚本采用统一的版本号体系：

#### 版本号格式：`V[major]`
- **V1（初始版本）**：仅添加标签，不删除已存在的标签
- **V2（智能清理版本）**：添加标签并删除不匹配的标签，代码结构优化

**版本对应关系：**
| 版本 | 创建日期 | 基础版本 | 操作模式 | 删除标签 | 代码结构 | 主要特性 |
|------|---------|---------|---------|---------|---------|---------|
| V1 | 2025-10-26 9:39pm | - | 仅添加 | ❌ 无 | 简单结构 | 基础版本，只添加标签 |
| V2 | 2025-10-30 2:57am | V1 | 智能清理 | ✅ 有 | 优化结构（feMapping数组） | 添加智能清理功能，代码结构优化 |

### V 系列迭代历史

#### V1 - 初始版本（2025-10-26）
**核心特性：**
- 基础的标签添加功能
- 检查条目的 `A2-IV/` 和 `A6-CV/` 标签
- 匹配固定效应关键词（地区、年份、行业）
- 添加固定效应类型标签（/@省份/FE, /@年份/FE, /@行业/FE）
- 仅添加标签，不删除已存在的标签

**检测的前缀：**
- `A2-IV/`：自变量
- `A6-CV/`：控制变量

**固定效应类型映射（3种）：**
1. **地区固定效应**：
   - 关键词：`["省份", "城市", "地区"]`
   - 标签：`/@省份/FE`
2. **年份固定效应**：
   - 关键词：`["年份", "年度"]`
   - 标签：`/@年份/FE`
3. **行业固定效应**：
   - 关键词：`["行业", "产业"]`
   - 标签：`/@行业/FE`

**处理逻辑：**
1. 检查条目是否有 `A2-IV/` 或 `A6-CV/` 前缀的标签
2. 如果没有这些标签，跳过（noMatchCount++）
3. 如果有这些标签，分别检查三种固定效应关键词
4. 如果包含关键词且目标标签不存在，添加标签
5. 如果添加了标签，保存条目（successCount++）
6. 如果有目标标签但没有匹配的关键词，跳过（skipCount++）

**统计信息：**
- 处理成功（successCount）
- 跳过（skipCount）
- 无匹配（noMatchCount）
- 错误（errorCount）
- 添加统计（按固定效应类型）

**代码结构：**
- 使用三个独立的关键词列表（locationKeywords, yearKeywords, industryKeywords）
- 三个独立的检查逻辑（hasLocationKeyword, hasYearKeyword, hasIndustryKeyword）

**特点：**
- ✅ 基础功能完整
- ✅ 支持3种固定效应类型（地区、年份、行业）
- ❌ 代码结构较为冗余（三个独立的检查逻辑）
- ❌ 不会删除不匹配的标签（标签可能过时）
- ❌ 无匹配条目会跳过（noMatchCount）

---


#### V2 - 智能清理版本（2025-10-30）
**核心改进：**
- 添加智能清理功能
- 如果标签中不包含关键词，会删除对应的固定效应类型标签
- 代码结构优化（使用 feMapping 数组）
- 移除 noMatchCount 统计
- 统计信息更加详细（添加和删除分别统计）

**代码结构优化：**
- **V1版本**：使用三个独立的关键词列表和检查逻辑
- **V2版本**：使用 feMapping 数组，统一处理逻辑

```javascript
// V1版本：三个独立的检查逻辑
const locationKeywords = ["省份", "城市", "地区"];
const yearKeywords = ["年份", "年度"];
const industryKeywords = ["行业", "产业"];

// V2版本：统一的映射数组
const feMapping = [
  {
    type: "地区固定效应",
    keywords: ["省份", "城市", "地区"],
    tag: "/@省份/FE",
    key: "location"
  },
  {
    type: "年份固定效应",
    keywords: ["年份", "年度"],
    tag: "/@年份/FE",
    key: "year"
  },
  {
    type: "行业固定效应",
    keywords: ["行业", "产业"],
    tag: "/@行业/FE",
    key: "industry"
  }
];
```

**处理逻辑变化：**
1. 检查条目是否有 `A2-IV/` 或 `A6-CV/` 前缀的标签
2. **不再跳过没有目标标签的条目**（移除 noMatchCount）
3. 遍历每种固定效应类型（使用 feMapping 数组）：
   - 如果包含关键词且目标标签不存在，添加标签
   - **如果 không 包含关键词但目标标签存在，删除标签**
4. 如果有任何修改，保存条目（processCount++）
5. 如果没有任何修改，跳过（skipCount++）

**统计信息变化：**
- 处理成功（processCount）
- 无变化（skipCount）
- 错误（errorCount）
- **添加标签统计**（按固定效应类型）
- **删除标签统计**（按固定效应类型）

**变化对比：**

| 特性 | V1 | V2 |
|------|----|----|
| 操作模式 | 仅添加 | 智能清理 |
| 删除标签 | ❌ 无 | ✅ 有 |
| noMatchCount | ✅ 有 | ❌ 无 |
| 统计信息 | 添加统计 | 添加统计 + 删除统计 |
| 代码结构 | 三个独立的检查逻辑 | feMapping 数组（统一处理） |
| 处理逻辑 | 有关键词才处理 | 所有条目都处理（删除不匹配的） |
| 标签维护 | ❌ 不会自动清理 | ✅ 自动清理不匹配的标签 |
| 代码可维护性 | 较低（冗余代码） | 较高（统一结构） |

**特点：**
- ✅ 添加智能清理功能（自动删除不匹配的标签）
- ✅ 代码结构优化（使用 feMapping 数组，统一处理逻辑）
- ✅ 标签维护更完善（标签与内容保持一致）
- ✅ 统计信息更详细（添加和删除分别统计）
- ✅ 代码可维护性更高（易于扩展新的固定效应类型）
- ✅ 所有条目都会处理（不再跳过没有目标标签的条目）

- **核心变化**：添加智能清理功能，代码结构优化

### 技术亮点

#### 1. 固定效应类型覆盖
- 支持3种常见的固定效应类型（地区、年份、行业）
- 符合面板数据分析的常见需求


#### 2. 关键词匹配
- 支持多个关键词匹配
- 地区固定效应：`["省份", "城市", "地区"]`
- 年份固定效应：`["年份", "年度"]`
- 行业固定效应：`["行业", "产业"]`
- 关键词匹配使用 `includes` 方法（部分匹配）


#### 3. 标签格式规范
- 统一使用 `/@XXX/FE` 格式
- 地区固定效应：`/@省份/FE`
- 年份固定效应：`/@年份/FE`
- 行业固定效应：`/@行业/FE`


#### 4. 代码结构优化（V2）
- 使用 feMapping 数组，统一处理逻辑
- 提高代码可维护性
- 易于扩展新的固定效应类型


#### 5. 智能清理（V2）
- 自动删除不匹配的标签
- 保持标签与内容一致
- 减少人工维护成本

---


### 注意事项
1. **版本选择：** V1 不会删除已存在的标签，V2 会删除不匹配的标签
2. **标签格式：** 所有版本使用统一的标签格式（/@XXX/FE）
3. **关键词匹配：** 使用部分匹配（`includes`），关键词不需要完全一致
4. **固定效应类型：** 支持3种常见的固定效应类型，如需添加新的类型，V2 版本更容易扩展
5. **标签维护：** V2 会自动清理不匹配的标签，建议定期运行以保持标签一致性
6. **统计信息：** V1 有 noMatchCount 统计，V2 移除该统计，所有条目都会处理
7. **代码结构：** V2 版本代码结构更优，使用 feMapping 数组，易于维护和扩展
