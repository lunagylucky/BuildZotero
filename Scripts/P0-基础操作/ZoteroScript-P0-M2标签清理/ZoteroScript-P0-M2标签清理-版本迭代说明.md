## M2- 标签清理 版本迭代说明

### 概述
M2- 标签清理模块用于批量清理条目中不符合规范的标签，通过定义允许保留的标签前缀列表，删除以字母或数字开头但不符合规范前缀的标签。


### 版本迭代历史

#### V1 - 初始版本（2025-08-03）
**允许保留的前缀（5 个）：**

```javascript
const allowedPrefixes = ["/", "Item/", "Method/", "Sample/", "Theory/"];
```

**核心特性：**
- 基础的五维标签保护
- 删除以字母或数字开头且不符合允许前缀的标签
- 保留特殊标记标签（`/` 前缀）
- 批量处理选中的条目
- 提供统计信息（处理条目数、删除标签数、失败条目数）

**删除规则：**
- 删除以字母或数字开头的标签
- 但保留符合允许前缀的标签
- 不删除以特殊符号（emoji、中文等）开头的标签

---


#### V2 - 方法标签标准化（2025-10-12）
**允许保留的前缀（5 个）：**

```javascript
const allowedPrefixes = ["/", "Item/", "sMeth/", "Sample/", "Theory/"];
```

**核心改进：**
- 将 `Method/` 改为 `sMeth/`，标准化方法标签前缀
- 保持其他功能不变
- 保持 5 个允许前缀的数量

**变化对比：**  

| 特性 | V1 | V2 |
|------|----|----|
| 方法标签前缀 | `Method/` | `sMeth/` |
| 允许前缀总数 | 5 个 | 5 个 |
| 其他功能 | 相同 | 相同 |

**原因说明：**
- 统一标签命名规范，将方法标签前缀标准化为 `sMeth/`

---


#### V3 - 扩展标签体系（2025-10-13）
**允许保留的前缀（14 个）：**

```javascript
const allowedPrefixes = [
  "/", 
  "Item/", 
  "sMeth/",           // 研究方法标签
  "Sample/", 
  "Result/",          // 新增：结果标签
  "Theory/",
  "Scholar/",         // 新增：学者标签
  "A1-DV/",           // 新增：因变量标签
  "A2-IV/",           // 新增：自变量标签
  "A3-MO/",           // 新增：调节变量标签
  "A4-ME/",           // 新增：中介变量标签
  "A5-INV/",          // 新增：工具变量标签
  "A6-CV/"            // 新增：控制变量标签
];
```

**核心改进：**
- 增加 Result/ 结果标签
- 增加 Scholar/ 学者标签
- 增加 A 系列变量标签（6 个）：A1-DV/, A2-IV/, A3-MO/, A4-ME/, A5-INV/, A6-CV/
- 从 5 个允许前缀扩展到 14 个
- 保护更完整的标签体系

**新增保护类别：**
- 基础标签：Result/（结果标签）、Scholar/（学者标签）
- A 系列变量标签（6 个）：完整覆盖所有变量类型

**标签体系结构：**

```
基础标签（7类）：
├── /          特殊标记
├── Item/      主题标签
├── sMeth/     研究方法标签
├── Sample/    样本标签
├── Result/    结果标签
├── Theory/    理论标签
└── Scholar/   学者标签

A系列变量标签（6类）：
├── A1-DV/     因变量
├── A2-IV/     自变量
├── A3-MO/     调节变量
├── A4-ME/     中介变量
├── A5-INV/    工具变量
└── A6-CV/     控制变量
```

### 版本对比总结
| 版本 | 创建日期 | 允许前缀数量 | 新增保护类别 | 主要变化 |
|------|---------|------------|------------|---------|
| V1 | 2025-08-03 | 5 个 | 基础五维标签 | 初始版本 |
| V2 | 2025-10-12 | 5 个 | - | Method/ → sMeth/ 标准化 |
| V3 | 2025-10-13 | 14 个 | Result/, Scholar/, A 系列 (6 个) | 扩展变量标签体系 |
| V4 | 2025-10-15 | 18 个 | V 系列 (4 个) | 完整标签体系保护 |


### 技术演进路径
1. **V1 → V2：标准化改进**
   - 方法标签前缀标准化（Method/ → sMeth/）
   - 保持向后兼容性

2. **V2 → V3：体系扩展**
   - 增加结果标签和学者标签
   - 引入 A 系列变量标签（6 个变量类型）
   - 从 5 个扩展到 14 个保护前缀

3. **V3 → V4：体系完善**
   - 增加 V 系列条目信息标签（4 个研究要素）
   - 从 14 个扩展到 18 个保护前缀
   - 形成完整的标签保护体系


### 使用建议
- **V1/V2 适用场景：** 只需要基础五维标签保护的小型项目
- **V3 适用场景：** 需要保护变量标签体系的实证研究项目
- **V4 适用场景：** 需要完整标签体系保护的研究项目（**推荐使用**）

**推荐版本：** V4（最完整的标签保护体系）


### 注意事项
1. **版本兼容性：** 每个新版本都向后兼容，可以安全地从旧版本升级
2. **标签规范：** 所有版本都遵循统一的标签命名规范
3. **性能影响：** 允许前缀数量增加不会显著影响性能，因为使用 `startsWith()` 方法快速匹配
