## M4- 标题清理 版本迭代说明

### 概述
M4- 标题清理模块用于批量清理条目标题中的 HTML 标签，移除常见的 HTML 格式化标签（如斜体标签、scp 标签等），并清理多余空格，保持标题的简洁性。


### 版本迭代历史

#### V1 - 初始版本（2025-08-10）
**核心功能：**  
清理条目标题中的 HTML 标签，移除格式化标记。

**清理的 HTML 标签：**

```javascript
const tagsToRemove = [
  /<i>/gi,           // 斜体开始标签
  /<\/i>/gi,         // 斜体结束标签
  /<scp>/gi,         // scp开始标签
  /<\/scp>/gi        // scp结束标签
];
```

**处理流程：**
1. 遍历选中的条目
2. 获取原始标题
3. 逐个清理指定的 HTML 标签
4. 清理多余空格（多个空格合并为一个，去除首尾空格）
5. 如果标题有变化，保存修改后的条目

**核心特性：**
- 批量处理选中的条目
- 使用正则表达式全局匹配（/gi 标志）
- 逐标签清理，记录变更状态
- 自动清理多余空格
- 只保存有变更的条目

**清理示例：**

```
原始标题: "<i>数字化转型</i>对企业绩效的影响"
清理后: "数字化转型对企业绩效的影响"

原始标题: "<scp>Resource</scp>-based view"
清理后: "Resource-based view"

原始标题: "标题  有   多个    空格"
清理后: "标题 有 多个 空格"
```

**统计信息：**
- 处理的条目数（成功修改的条目）
- 修改的标题总数
- 失败条目数

**错误处理：**
- 单个条目处理失败不影响其他条目
- 保存失败单独统计

**使用场景：**
- 批量导入文献后清理标题中的 HTML 标签
- 修复从其他系统导入的格式问题
- 保持标题的简洁性和可读性

### 技术实现要点

#### 正则表达式使用

```javascript
/<i>/gi           // g: 全局匹配，i: 忽略大小写
/<\/i>/gi         // 转义的结束标签
```


#### 变更检测

```javascript
const beforeClean = cleanedTitle;
cleanedTitle = cleanedTitle.replace(tagPattern, '');
if (beforeClean !== cleanedTitle) {
  hasChanges = true;
}
```

- 通过比较清理前后字符串检测变更
- 确保只保存有实际变更的条目


#### 空格清理

```javascript
cleanedTitle = cleanedTitle.replace(/\s+/g, ' ').trim();
```

- `/\s+/g`：匹配一个或多个空白字符（空格、制表符等）
- 替换为单个空格
- `trim()`：去除首尾空格


#### 事务保存

```javascript
await item.saveTx();
```

- 使用事务保存确保数据一致性
- 异步处理提升性能

### 使用建议
**适用场景：**
- ✅ 批量清理导入文献的标题格式
- ✅ 修复 HTML 标签残留问题
- ✅ 统一标题格式规范
- ✅ 提升标题可读性

**注意事项：**
1. **操作前备份：** 虽然操作可逆（通过修改），但建议操作前备份数据
2. **批量测试：** 首次使用时建议选择少量条目测试效果
3. **检查结果：** 清理后检查标题是否符合预期
4. **特殊格式：** 如果标题中包含需要保留的 HTML 格式，需要手动处理

**最佳实践：**
1. 先选择 1-2 个条目测试
2. 确认清理效果符合预期
3. 再批量处理所有需要清理的条目
4. 检查统计信息，确认处理结果
