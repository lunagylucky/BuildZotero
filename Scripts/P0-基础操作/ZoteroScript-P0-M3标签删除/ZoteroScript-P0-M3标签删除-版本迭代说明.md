## M3- 标签删除 版本迭代说明

### 概述
M3- 标签删除模块用于批量删除条目中指定前缀或类型的标签，提供安全预览、确认机制和详细的统计信息，支持多种输入方式（前缀、数字选项、名称映射等）。


### 版本迭代历史

#### V1 - 基础版本（2025-08-09）
**核心特性：**
- 单前缀删除：支持输入一个标签前缀进行删除
- 双重确认机制：输入确认 + 预览确认
- 预览功能：显示前 5 个条目和将要删除的标签
- 详细统计：处理条目数、删除标签数、跳过条目数、错误数

**功能特点：**

```javascript
// 用户输入
const userInput = prompt("请输入要删除的标签前缀（例如：Result/ 或 Theory/ 或 Sample/1宏观）:");

// 双重确认
1. 输入后立即确认
2. 预览后再次确认
```

**安全机制：**
- 预览匹配的标签列表
- 显示将要删除的标签数量
- 支持操作取消

**统计信息：**
- 处理条目数
- 删除标签数
- 跳过条目数（无匹配标签）
- 错误数

---


#### V2 - 多前缀支持（2025-10-12）
**核心改进：**
- 支持多前缀删除：逗号分隔的多个前缀
- 解析多个前缀输入
- 统一确认对话框显示所有前缀
- 预览功能支持多前缀匹配

**功能特点：**

```javascript
// 用户输入（支持多个）
const userInput = prompt("请输入要删除的标签前缀，多个前缀用逗号分隔\n例如：sMeth/, Theory/, Sample/,  Item/, Result/,Sample/1宏观");

// 解析多个前缀
const tagPrefixes = userInput.split(',').map(prefix => prefix.trim()).filter(prefix => prefix !== "");
```

**新增功能：**
- 批量前缀删除：一次操作可删除多种类型的标签
- 灵活的输入方式：支持空格、逗号分隔
- 智能过滤：自动去除空字符串

**改进点：**
- 确认对话框显示所有输入的前缀
- 预览信息包含所有匹配的前缀
- 统计信息显示所有前缀的删除结果

**应用场景：**
- 一次性清理多个类型的标签
- 批量删除测试标签
- 清理多个相关标签类别

---


#### V3 - 智能映射系统（2025-10-15）
**核心改进：**  
这是功能最全面的版本，引入了完整的标签映射系统。

**新增功能：**

1. **数字选项映射系统**

```javascript
const tagMapping = {
  "1": ["A1-DV/", "A2-IV/", "A3-MO/", "A4-ME/", "A5-INV/", "A6-CV/"],
  "1-1": "A1-DV/", "1-2": "A2-IV/", ...
  "2": "Item/",
  "3": "Theory/",
  ...
};
```

2. **名称映射系统（中英文支持）**

```javascript
const nameMapping = {
  "因变量": "A1-DV/", "dv": "A1-DV/",
  "理论": "Theory/", "theory": "Theory/",
  "变量标签": ["A1-DV/", "A2-IV/", ...],
  ...
};
```

3. **精确标签匹配**
- 支持完整标签名匹配（如：`Theory/自我决定理论`）
- 支持按核心名称匹配（忽略前缀、序号、-new 后缀）

4. **核心名称标准化**

```javascript
const normalizeCore = (input) => {
  // 去除前缀、序号、-new后缀
  // 例如："Theory/自我决定理论-new" → "自我决定理论"
};
```

5. **查询模式**
- 新增查询模式：仅查询匹配的标签而不执行删除
- 可以预览将要删除的标签而不实际删除

**输入方式支持：**
- 数字选项：`1,3,4` 或 `1-1,1-2`
- 名称输入：`因变量, 理论标签, 变量定义`
- 前缀输入：`Theory/, Item/`
- 完整标签：`Theory/自我决定理论, Sample/1宏观`
- 混合输入：`1,Theory/,变量定义,Sample/1宏观`

**用户交互改进：**
- 详细的提示信息，展示所有可用选项
- 智能解析用户输入（数字/名称/前缀/完整标签）
- 显示解析后的实际前缀和精确标签
- 查询模式确认对话框

**统计信息增强：**
- 显示前缀列表和精确标签列表
- 区分前缀匹配和精确匹配的结果

---


#### V4 - 特殊模式支持（2025-10-17）
**核心改进：**  
在 V3 的基础上增加特殊模式 0，专门用于删除 `Theory/*-new` 类型的标签。

**新增功能：**

1. **特殊模式 0**

```javascript
// 仅删除 Theory/*-new 标签
const theoryNewOnly = userChoices.length === 1 && userChoices[0] === "0";

// 特殊匹配函数
const shouldDelete = (tagName) => {
  if (theoryNewOnly) {
    return tagName.startsWith("Theory/") && /-new$/i.test(tagName);
  }
  // ... 原有逻辑
};
```

2. **优化匹配逻辑**
- 提取 `shouldDelete` 函数，统一匹配逻辑
- 预览和删除使用相同的匹配函数，确保一致性

3. **用户输入改进**
- 在提示信息中添加模式 0 的说明
- 优化模式 0 的处理流程

**应用场景：**
- 清理临时标记的新理论标签（Theory/*-new）
- 批量删除特定格式的测试标签
- 维护理论标签的版本控制

**模式对比：**  

| 模式 | 输入 | 匹配规则 |
|------|------|---------|
| 普通模式 | `1,3,4` 等 | 根据映射系统匹配 |
| 特殊模式 0 | `0` | 仅匹配 `Theory/` 开头且以 `-new` 结尾的标签 |

**统计信息增强：**
- 显示当前模式（普通模式/Theory/*-new 专用）
- 区分不同模式的删除结果

### 功能演进路径

#### 1. V1 → V2：批量处理能力
- 从单前缀扩展到多前缀
- 提升批量删除效率


#### 2. V2 → V3：智能化升级
- 引入数字选项系统（用户友好）
- 引入名称映射系统（中英文支持）
- 支持精确标签匹配
- 新增查询模式（安全预览）
- 核心名称标准化（智能匹配）


#### 3. V3 → V4：特殊场景优化
- 增加特殊模式 0（Theory/*-new 专用）
- 优化匹配逻辑（统一函数）
- 改进统计信息（模式区分）


### 使用建议

#### 选择版本指南
- **V1 适用场景：**
  - 简单的前缀删除需求
  - 只需要基础功能

- **V2 适用场景：**
  - 需要批量删除多个前缀
  - 不需要复杂的映射系统

- **V3 适用场景（推荐）：**
  - 需要灵活的输入方式
  - 需要查询预览功能
  - 需要精确标签匹配
  - 需要完整的映射系统

- **V4 适用场景（最新推荐）：**
  - 需要删除 Theory/*-new 标签
  - 需要最优化的匹配逻辑
  - 需要最新的功能特性

**推荐版本：** V4（功能最完整，性能最优）


### 技术亮点

#### V3/V4 的核心技术
1. **智能输入解析**
   - 自动识别数字选项、名称、前缀、完整标签
   - 统一转换为标准前缀或精确标签

2. **核心名称标准化**

   ```javascript
   // 去除前缀、序号、-new后缀
   "Theory/1-自我决定理论-new" → "自我决定理论"
   ```

3. **双重匹配机制**
   - 前缀匹配：快速匹配整个类别
   - 核心匹配：精确匹配具体标签

4. **安全查询模式**
   - 预览而不删除
   - 确认后再执行

5. **统一匹配函数（V4）**
   - 预览和删除使用相同逻辑
   - 确保一致性


### 注意事项
1. **操作不可撤销：** 删除操作一旦执行无法撤销，建议先用查询模式预览
2. **匹配规则：** V3/V4 支持多种匹配方式，注意理解匹配逻辑
3. **性能考虑：** 大量条目处理时，V4 的统一匹配函数性能更优
4. **版本兼容：** 每个新版本都兼容旧版本的输入方式
