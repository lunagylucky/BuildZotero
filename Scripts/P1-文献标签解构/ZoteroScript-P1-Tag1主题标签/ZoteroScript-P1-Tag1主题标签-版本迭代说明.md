## Tag1- 主题标签 版本迭代说明

### 概述
Tag1- 主题标签模块用于从学术文献中自动提取和生成主题标签，通过 AI 分析文献内容（摘要、PDF、笔记）生成最能代表核心内容的标签。标签聚焦于核心变量、研究主题、研究结论和核心概念，避免与研究方法、理论名称、研究对象等标签重复。


### 版本号体系说明
主题标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major].[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定内容获取策略和 PDF 字符数
  - V1-V5：主要版本迭代

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）
  - **-Rnote**：笔记优先版本（删除旧标签后重新生成，优先使用笔记内容，需要 📒 标签，基于对应的 V 版本）

**R 系列与 Rnote 系列的区别：**
- **相同点**：都会删除现有的 Item/ 标签后重新生成
- **不同点**：
  - **R 系列**：内容获取策略与对应的 V 版本相同（如 V5-R 使用 PDF 前 8000 字符 → 摘要）
  - **Rnote 系列**：优先使用笔记内容，备选 PDF 或摘要（如 V5-Rnote 使用 笔记 → 摘要 → PDF 前 8000 字符）


#### 版本对应关系
| 基础版本 | 重新处理版本（R 系列） | 笔记优先版本（Rnote 系列） | 功能特性 |
|---------|---------------------|-------------------------|---------|
| V1 | - | - | 摘要优先，PDF 全文，无序号标签，无跳过机制 |
| V2 | - | - | 摘要优先，PDF 前 3000 字符，无序号标签，跳过机制 |
| V3 | - | - | PDF 前 3000 字符优先，无序号标签 |
| V4 | V4-R | V4-Rnote | PDF 前 3000 字符，序号标签，优先级明确 |
| V5 | V5-R | V5-Rnote | PDF 前 8000 字符，序号标签 |

**命名逻辑说明：**
- **V4-R**：基于 V4 的重新处理版本（在 V4 基础上增加删除旧标签功能）
- **V5-R**：基于 V5 的重新处理版本（在 V5 基础上增加删除旧标签功能）
- **V4-Rnote**：基于 V4 的笔记优先版本（在 V4 基础上增加笔记优先逻辑）
- **V5-Rnote**：基于 V5 的笔记优先版本（在 V5 基础上增加笔记优先逻辑）

### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-07-04）
**核心特性：**
- 基础的主题标签生成功能
- 优先使用摘要，备选 PDF 全文
- 简单的 AI Prompt（英文）
- 无跳过机制（可能重复处理已有标签的条目）

**内容获取策略：**

```javascript
// 优先摘要
let abs = item.getField("abstractNote");
// 备选PDF全文
if (!abs) {
  const pdfItem = await item.getBestAttachment();
  if (pdfItem) {
    abs = (await Zotero.PDFWorker.getFullText(pdfItem.id, 1)).text;
  }
}
```

**标签格式：**
- `Item/tag`（不带序号）
- 例如：`Item/创新绩效`, `Item/数字化转型`

**AI Prompt（英文）：**

```
"Read the abstract of the article below:\n" + abs +
"\n---\nReturns 3 tags that fit this abstract as a JSON list. 
Each tag should be in simplified Chinese and as concise as possible."
```

**特点：**
- ✅ 基础功能完整
- ❌ 无跳过机制，可能重复处理
- ❌ Prompt 较简单，未明确聚焦点
- ❌ 无内容处理不完善

---


#### V2 - 跳过机制与 Prompt 优化（2025-07-31）
**核心改进：**
- 添加跳过已处理条目机制
- 优化 AI Prompt，明确聚焦核心内容
- 完善无内容处理
- 优先摘要，备选 PDF 前 3000 字符

**新增功能：**
1. **跳过已处理条目**

```javascript
// 跳过已处理的条目
const existingTags = item.getTags();
if (existingTags.some(tag => tag.tag && tag.tag.startsWith("Item/"))) {
  skipCount++;
  continue;
}
```

2. **优化的内容获取策略**

```javascript
// 优先摘要（50字符阈值）
const abstract = item.getField("abstractNote");
if (abstract && abstract.trim().length > 50) {
  content = abstract.trim();
} else {
  // 备选PDF前3000字符
  content = fullTextData.text.substring(0, 3000).trim();
}
```

3. **优化的 AI Prompt（中文）**

```
分析以下学术文献内容，生成3个最能代表其核心内容的标签：
要求：
1. 标签应简洁精确，3-6个字符
2. 使用简体中文
3. 重点关注：核心变量、研究主题、研究结论、核心概念
4. 避免研究方法、理论名称、研究对象（这些有单独标签）
5. 聚焦文献的核心贡献和发现
6. 避免过于宽泛的词汇，要具体明确
```

4. **无内容标记**

```javascript
item.addTag('Item/无摘要');
```

**标签格式：**
- `Item/tag`（不带序号）
- 例如：`Item/创新绩效`, `Item/数字化转型`

**变化对比：**

| 特性 | V1 | V2 |
|------|----|----|
| 跳过机制 | ❌ 无 | ✅ 有 |
| 内容优先级 | 摘要 > PDF 全文 | 摘要 > PDF 前 3000 字符 |
| Prompt 语言 | 英文 | 中文 |
| Prompt 详细度 | 简单 | 详细，明确聚焦点 |
| 无内容标记 | ❌ | ✅ `Item/无摘要` |
| 统计信息 | 基础 | 详细（成功/跳过/无内容/失败/错误） |

#### V3 - PDF 优先策略（2025-08-08）
**核心改进：**
- 调整内容获取优先级：优先 PDF，备选摘要
- 使用 PDF 前 3000 字符，包含更丰富的主题信息
- 无内容标记改为 `Item/无内容`

**内容获取策略变化：**

```javascript
// V2: 优先摘要
abstract > PDF前3000字符

// V3: 优先PDF
PDF前3000字符 > abstract
```

**原因说明：**
- PDF 内容更全面，包含摘要、引言、部分结果和结论
- 3000 字符包含更多主题相关信息
- 平衡信息量与处理效率

**标签格式：**
- `Item/tag`（不带序号）
- 与 V2 相同

**无内容标记：**
- `Item/无内容`（从 `Item/无摘要` 改为更通用的标记）

**其他特性：**
- Prompt 与 V2 相同
- 跳过机制与 V2 相同

#### V4 - 序号标签与 Prompt 优先级（2025-10-12）
**核心改进：**
- 标签格式增加序号：`Item/1tag`, `Item/2tag`, `Item/3tag`
- Prompt 明确优先级：第一优先级是变量名称，第二优先级是关键词
- 保持 V3 的内容获取策略

**标签格式变化：**

```javascript
// V3: 无序号
item.addTag('Item/' + tag);

// V4: 带序号
item.addTag('Item/1' + validTags[0]);
if (validTags.length > 1) item.addTag('Item/2' + validTags[1]);
if (validTags.length > 2) item.addTag('Item/3' + validTags[2]);
```

**标签示例：**
- V3: `Item/创新绩效`, `Item/数字化转型`, `Item/正向影响`
- V4: `Item/1创新绩效`, `Item/2数字化转型`, `Item/3正向影响`

**Prompt 优化：**

```
分析以下学术文献内容，生成3个最能代表其核心内容的标签,
第一优先级是变量名称，第二优先级是关键词：
...
3. 重点关注：核心变量、核心概念为主、关键词是第二等级，关系结论是最后考虑
...
6. 按照相关程度排序返回，第1个标签最相关，第3个标签相关度最低
```

**序号标签的优势：**
- 明确标签的重要性顺序
- 便于按重要性筛选和排序
- 支持优先使用最重要的标签

**变化对比：**

| 特性 | V3 | V4 |
|------|----|----|
| 标签格式 | `Item/tag` | `Item/1tag`, `Item/2tag`, `Item/3tag` |
| Prompt 优先级 | 未明确 | ✅ 明确：变量名称 > 关键词 > 关系结论 |
| 标签排序 | 未强调 | ✅ 强调按相关程度排序 |

---


#### V5 - 扩展 PDF 内容长度（2025-10-13）
**核心改进：**
- PDF 内容从 3000 字符扩展到 8000 字符
- 其他功能与 V4 相同

**内容获取策略变化：**

```javascript
// V4: PDF前3000字符
content = fullTextData.text.substring(0, 3000).trim();

// V5: PDF前8000字符
content = fullTextData.text.substring(0, 8000).trim();
```

**原因说明：**
- 更长的文本包含更多上下文信息
- 有助于 AI 更准确地理解文献核心内容
- 能够提取更丰富和准确的主题标签

**性能考虑：**
- 文本长度增加会增加 AI 处理时间
- 但能提高标签质量和准确性
- 平衡质量与效率

**标签格式：**
- 与 V4 相同：`Item/1tag`, `Item/2tag`, `Item/3tag`

**Prompt：**
- 与 V4 相同

**变化对比：**

| 特性 | V4 | V5 |
|------|----|----|
| PDF 字符数 | 3000 | 8000 |
| 处理速度 | 较快 | 较慢 |
| 标签质量 | 良好 | 更优 |

### R 系列（重新处理版本）迭代历史
**系列说明：**
- **R** = Re-process（重新处理）
- R 系列版本会：
  1. **删除现有的 Item/ 标签后重新生成**
  2. 使用与对应 V 版本相同的内容获取策略（如 V5-R 使用 PDF 前 8000 字符 → 摘要）
  3. 处理所有条目（不需要 📒 标签）

---

#### V4-R - 基于 V4 的重新处理版本（2025-10-12）
**命名说明：** `V4-R` 表示这是**基于 V4 版本的重新处理版本**。在 V4 的基础上增加了删除旧标签后重新生成的功能。

**核心特性：**
- **删除现有 Item/ 标签后重新生成**（与 Rnote 系列相同）
- 优先 PDF 前 3000 字符，备选摘要（与 V4 相同）
- 标签格式：`Item/1tag`, `Item/2tag`, `Item/3tag`（带序号，与 V4 相同）
- Prompt 与 V4 相同，强调变量名称优先级

**删除机制：**

```javascript
const removeExistingItemTags = async (item) => {
  const existingTags = item.getTags();
  const itemTags = existingTags.filter(tag => tag.tag && tag.tag.startsWith("Item/"));
  for (let itemTag of itemTags) {
    item.removeTag(itemTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingItemTags(item);
```

**内容获取策略：**
- 优先 PDF 前 3000 字符
- 备选摘要

**标签格式：**
- `Item/1tag`, `Item/2tag`, `Item/3tag`（带序号）

**Prompt：**
- 与 V4 相同，强调变量名称优先级

**与 V4 的区别：**

| 特性 | V4 | V4-R |
|------|----|------|
| 跳过机制 | ✅ 跳过已有标签的条目 | ❌ 不跳过，删除旧标签 |
| 删除机制 | ❌ 无 | ✅ 删除现有 Item/ 标签 |
| 适用场景 | 新增标签 | 更新/重新生成标签 |

**应用场景：**
- 需要更新已有主题标签
- Prompt 优化后重新生成标签
- 修正错误的标签

### Rnote 系列（笔记优先版本）迭代历史
**系列说明：**
- **Rnote** = Re-process（重新处理）+ Note-priority（笔记优先）
- Rnote 系列版本会：
  1. **删除现有的 Item/ 标签后重新生成**（与 R 系列相同）
  2. **检查 📒 标签**，仅处理带有 📒 标签的条目
  3. **优先使用笔记内容**，备选 PDF 或摘要
- **与 R 系列的区别**：
  - **相同点**：都会删除旧标签后重新生成
  - **不同点**：Rnote 系列优先使用笔记内容，而 R 系列使用与对应 V 版本相同的内容获取策略

---

#### V4-Rnote - 基于 V4 的笔记优先版本（2025-10-12）
**命名说明：** `V4-Rnote` 表示这是**基于 V4 版本的笔记优先版本**。在 V4 的基础上增加了笔记优先逻辑（检查 📒 标签，优先获取笔记内容），同时会删除旧标签后重新生成（与 R 系列相同）。

**核心特性：**
- **删除现有 Item/ 标签后重新生成**（与 R 系列相同）
- **仅处理带有 📒 标签的条目**
- **优先使用笔记内容**（Zotero Notes，转换为 Markdown）
- 备选摘要，然后 PDF 前 3000 字符
- 标签格式：`Item/1tag`, `Item/2tag`, `Item/3tag`（带序号，与 V4 相同）
- Prompt 与 V4 相同

**笔记检测机制：**

```javascript
const hasNotebookTag = (item) => {
  const tags = item.getTags();
  return tags.some(tag => tag.tag && tag.tag.includes("📒"));
};

// 只有有📒标签的条目才处理
if (!hasNotebookTag(item)) {
  skipCount++;
  continue;
}
```

**内容获取优先级：**

```javascript
// 第一优先：笔记内容（BetterNotes Markdown）
const noteContent = await Zotero.BetterNotes.api.convert.html2md(noteItem.getNote());

// 第二优先：摘要
const abstract = item.getField("abstractNote");

// 第三优先：PDF前3000字符
content = fullTextData.text.substring(0, 3000).trim();
```

**与 V4-R 的区别：**

| 特性 | V4-R | V4-Rnote |
|------|------|----------|
| 删除旧标签 | ✅ 是 | ✅ 是（相同） |
| 内容获取策略 | PDF 前 3000 字符 → 摘要 | 笔记 → 摘要 → PDF 前 3000 字符（优先笔记） |
| 📒 标签要求 | ❌ 无 | ✅ 必须有 |
| 适用场景 | 所有条目 | 仅已阅读并做笔记的条目 |

**与 V4 的区别：**

| 特性 | V4 | V4-Rnote |
|------|----|----------|
| 条目筛选 | 跳过已有标签 | 仅处理📒标签条目 |
| 删除旧标签 | ❌ 无 | ✅ 是（与 R 系列相同） |
| 内容优先级 | PDF > 摘要 | 笔记 > 摘要 > PDF |
| 笔记支持 | ❌ | ✅ BetterNotes 转换 |
| 适用场景 | 所有条目 | 已阅读并做笔记的文献 |

**笔记内容优势：**
- 笔记是用户提炼和总结的内容
- 包含用户认为重要的核心信息
- 通常已经过人工筛选，质量更高

**标签格式：**
- `Item/1tag`, `Item/2tag`, `Item/3tag`（带序号）

**Prompt：**
- 与 V4 相同

**应用场景：**
- 已阅读并做笔记的文献
- 需要基于笔记内容生成主题标签
- 利用用户提炼的信息提高标签准确性

### 使用建议

#### 选择版本指南

##### V 系列（标准版本）适用场景
- **V1：** 不推荐使用（功能不完善）
- **V2：** 适合摘要丰富的文献，需要中文 Prompt
- **V3：** 适合 PDF 丰富的文献，PDF 优先策略
- **V4：** **推荐** - 需要序号标签，明确优先级排序
- **V5：** **推荐** - 需要最高标签质量，可接受较长处理时间


##### R 系列（重新处理版本）适用场景
- **V4-R：** 需要更新标签，使用较短 PDF 内容（3000 字符）
- **V5-R：** **推荐** - 需要更新标签，追求更高质量（8000 字符）


##### Rnote 系列（笔记优先版本）适用场景
- **V4-Rnote：** 已做笔记的文献，使用较短 PDF 后备（3000 字符）
- **V5-Rnote：** **推荐** - 已做笔记的文献，追求更高质量（8000 字符）

### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本系列
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，V5/V5-R/V5-Rnote 处理时间较长
4. **标签格式：** V4/V5/R/Rnote 系列使用序号标签，与早期版本不兼容
5. **📒 标签：** Rnote 系列需要条目有 📒 标签才会处理
6. **重复处理：** R 系列会删除旧标签，请确认是否需要保留旧标签
7. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
