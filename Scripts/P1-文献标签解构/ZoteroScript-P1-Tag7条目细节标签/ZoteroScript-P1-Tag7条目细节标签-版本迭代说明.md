## Tag7- 条目细节标签 版本迭代说明

### 概述
Tag7- 条目细节标签模块用于从学术文献中自动识别和提取研究的详细要素标签，通过 AI 分析文献内容（摘要、PDF、笔记）识别 5 类关键信息：变量定义（V1-def）、测量方法（V2-mea）、研究贡献（V3-con）、研究展望（V4-fut）和样本与数据来源（V5-sor）。标签体系采用 `V1-def/`, `V2-mea/`, `V3-con/`, `V4-fut/`, `V5-sor/` 前缀，便于分类管理和检索。


### 版本号体系说明
条目细节标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major].[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定标签类别数量（4类或5类）、内容获取策略、PDF 字符数和 Prompt 优化程度
  - V1：4 类标签（变量定义、测量方法、研究贡献、研究展望）
  - V2：5 类标签（在 V1 基础上新增样本与数据来源）

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（检查并跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）
  - **-Rnote**：笔记优先版本（删除旧标签后重新生成，优先使用笔记内容，需要 📒 标签，基于对应的 V 版本）

**R 系列与 Rnote 系列的区别：**
- **相同点**：都会删除现有的 `V1-def/` 到 `V5-sor/` 标签后重新生成
- **不同点**：
  - **R 系列**：内容获取策略与对应的 V 版本相同（如 V2-R 使用 PDF 前 45000 字符 → 摘要）
  - **Rnote 系列**：优先使用笔记内容，备选 PDF 或摘要（如 V1-Rnote 使用 笔记 → PDF 前 45000 字符 → 摘要）


#### 版本对应关系
| 基础版本 | 重新处理版本（R 系列） | 笔记优先版本（Rnote 系列） | 功能特性 |
|---------|---------------------|-------------------------|---------|
| V1 | V1-R | V1-Rnote | 4 类标签（变量定义、测量方法、研究贡献、研究展望），PDF 前 45000 字符优先，摘要备选 |
| V2 | V2-R | V2-Rnote | 5 类标签（在 V1 基础上新增样本与数据来源），PDF 前 45000 字符优先，摘要备选 |

**命名逻辑说明：**
- **V1-R**：基于 V1 的重新处理版本（在 V1 基础上增加删除旧标签功能）
- **V2-R**：基于 V2 的重新处理版本（在 V2 基础上增加删除旧标签功能）
- **V1-Rnote**：基于 V1 的笔记优先版本（在 V1 基础上增加笔记优先逻辑）
- **V2-Rnote**：基于 V2 的笔记优先版本（在 V2 基础上增加笔记优先逻辑）

### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-09-21）
**核心特性：**
- 基础的条目细节标签生成功能
- 4 类标签：变量定义（V1-def）、测量方法（V2-mea）、研究贡献（V3-con）、研究展望（V4-fut）
- 优先使用 PDF 前 45000 字符，备选摘要
- 检查并跳过已有标签的条目机制
- 使用 `saveItemWithTags` 函数进行事务保存

**标签类别：**
1. **变量定义（V1-def）**：识别文献中主要变量的学术定义或概念界定
2. **测量方法（V2-mea）**：识别文献中变量的具体测量方式或操作化方法
3. **研究贡献（V3-con）**：识别文献的主要学术贡献、理论创新、实证发现（1-4 个）
4. **研究展望（V4-fut）**：识别未来研究方向、研究建议、研究局限（1-3 个）

**内容获取策略：**

```javascript
// 优先PDF前45000字符
const pdfItem = await item.getBestAttachment();
if (pdfItem && pdfItem.isPDFAttachment()) {
  const fullTextData = await Zotero.PDFWorker.getFullText(pdfItem.id, 1);
  if (fullTextData && fullTextData.text && fullTextData.text.trim().length > 100) {
    content = fullTextData.text.substring(0, 45000).trim();
  }
}

// 备选摘要
if (!content) {
  const abstract = item.getField("abstractNote");
  if (abstract && abstract.trim().length > 50) {
    content = abstract.trim();
  }
}
```

**标签格式：**
- `V1-def/变量名:定义`（变量定义）
- `V2-mea/变量名:测量方式`（测量方法）
- `V3-con/贡献内容`（研究贡献，1-4 个）
- `V4-fut/展望内容`（研究展望，1-3 个）

**跳过机制：**
- 检查是否存在完整的 4 类标签（V1-def, V2-mea, V3-con, V4-fut）
- 如果存在，则跳过处理

**删除机制：**
- 在处理前删除现有的 `V[1-4]-(def|mea|con|fut)/` 标签
- 确保重新生成时没有旧标签干扰

**AI Prompt 特点：**
- 详细说明 4 类信息的定义和识别要点
- 明确要求内容精简到 15-25 个字符以内
- 去掉"本研究"、"我们"、"文章"等主语
- 去掉"发现"、"表明"、"认为"等冗余动词
- 只保留核心学术内容

**标签数量要求：**
- **至少 4 类标签**（每类至少 1 个标签，即使为"无"）

**特点：**
- ✅ 基础功能完整
- ✅ 跳过机制，避免重复处理
- ✅ 使用事务保存，提高可靠性
- ✅ PDF 45000 字符，包含更多信息
- ✅ 标签类别清晰，便于检索
- ❌ 缺少数据来源信息（V2 中新增）

#### V2 - 新增数据来源类别（2025-10-29）
**核心改进：**
- 新增第 5 类标签：样本与数据来源（V5-sor）
- 标签类别从 4 类扩展到 5 类
- 检查机制从 4 类标签扩展到 5 类标签
- Prompt 增加数据来源的详细说明和要求

**标签类别变化：**
- V1：4 类标签（V1-def, V2-mea, V3-con, V4-fut）
- V2：5 类标签（V1-def, V2-mea, V3-con, V4-fut, **V5-sor**）

**新增标签类别：**
5. **样本与数据来源（V5-sor）**：识别文献使用的具体数据库名称、数据平台、文本来源渠道（1-3 个）
   - **重要：区分"数据来源"与"研究样本"**
     * 数据来源：数据库、网站、平台等数据获取渠道（这才是要识别的）
     * 研究样本：研究对象的描述，如"沪深A股上市公司"、"中国制造业企业"（不是数据来源）
   - **字符限制**：10-20 个字符
   - **示例**：
     * 数值型数据源：CSMAR数据库、Wind数据库、国泰安数据库、USPTO专利库、Compustat、中国工业企业数据库
     * 文本型数据源：巨潮资讯网、新浪财经网、Twitter、LinkedIn、Facebook
     * 文本+网站结合：巨潮资讯网年报、新浪财经新闻、公司官网公告、Twitter推文
     * 文本类型（次优）：企业年度报告、财经新闻报道、管理层讨论与分析

**内容获取策略：**
- 与 V1 相同：优先 PDF 前 45000 字符，备选摘要

**标签格式：**
- 与 V1 相同，新增 `V5-sor/数据来源名称`（1-3 个）

**跳过机制：**
- 检查是否存在完整的 5 类标签（V1-def, V2-mea, V3-con, V4-fut, V5-sor）
- 如果存在，则跳过处理

**删除机制：**
- 在处理前删除现有的 `V[1-5]-(def|mea|con|fut|sor)/` 标签
- 确保重新生成时没有旧标签干扰

**AI Prompt 优化：**
- 增加第 5 类"样本与数据来源"的详细说明
- 明确区分数据来源与研究样本
- 提供大量数据源示例（数值型、文本型、社交媒体等）
- 强调数据来源必须具体到可查找的程度

**标签数量要求：**
- **至少 5 类标签**（每类至少 1 个标签，即使为"无"）

**变化对比：**

| 特性 | V1 | V2 |
|------|----|----|
| 标签类别数量 | 4 类 | 5 类（新增 V5-sor） |
| 跳过检查 | 检查 4 类标签 | 检查 5 类标签 |
| 删除范围 | `V[1-4]-(def|mea|con|fut)/` | `V[1-5]-(def|mea|con|fut|sor)/` |
| 标签数量要求 | 至少 4 类标签 | 至少 5 类标签 |
| 数据来源信息 | ❌ 无 | ✅ 有（V5-sor） |
| Prompt 复杂度 | 基础 | 扩展（增加数据来源说明） |

**后续版本基础：**
- V2 成为后续变体版本（V2-R, V2-Rnote）的基础版本

### R 系列（重新处理版本）迭代历史
**系列说明：**
- **R** = Re-process（重新处理）
- R 系列版本会：
  1. **删除现有的 `V1-def/` 到 `V5-sor/` 标签后重新生成**
  2. 使用与对应 V 版本相同的内容获取策略（如 V2-R 使用 PDF 前 45000 字符 → 摘要）
  3. 处理所有条目（不需要 📒 标签）

**核心特性：**

- 至少 4 类标签（与 V1 相同）
- Prompt 与 V1 相同

**删除机制：**

```javascript
const removeExistingResearchTags = async (item) => {
  const existingTags = item.getTags();
  const researchTags = existingTags.filter(tag => 
    tag.tag && /^V[1-4]-(def|mea|con|fut)\//.test(tag.tag)
  );
  for (let researchTag of researchTags) {
    item.removeTag(researchTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingResearchTags(item);
```

**内容获取策略：**
- 与 V1 相同：优先 PDF 前 45000 字符，备选摘要

**标签类别：**

**Prompt：**
- 与 V1 相同

**与 V1 的区别：**

| 特性 | V1 | V1-R |

### Rnote 系列（笔记优先版本）迭代历史
**系列说明：**
- **Rnote** = Re-process（重新处理）+ Note-priority（笔记优先）
- Rnote 系列版本会：
  1. **删除现有的 `V1-def/` 到 `V5-sor/` 标签后重新生成**（与 R 系列相同）
  2. **检查 📒 标签**，仅处理带有 📒 标签的条目
  3. **优先使用笔记内容**，备选 PDF 或摘要
- **与 R 系列的区别**：
  - **相同点**：都会删除旧标签后重新生成
  - **不同点**：Rnote 系列优先使用笔记内容，而 R 系列使用与对应 V 版本相同的内容获取策略

#### V1-Rnote - 基于 V1 的笔记优先版本（2025-10-15）
**命名说明：** `V1-Rnote` 表示这是**基于 V1 版本的笔记优先版本**。在 V1 的基础上增加了笔记优先逻辑（检查 📒 标签，优先获取笔记内容），同时会删除旧标签后重新生成（与 R 系列相同）。

**核心特性：**
- **删除现有 `V[1-4]-(def|mea|con|fut)/` 标签后重新生成**（与 R 系列相同）
- **仅处理带有 📒 标签的条目**
- **优先使用笔记内容**（Zotero Notes，转换为 Markdown）
- 备选 PDF 前 45000 字符，然后摘要
- 4 类标签：变量定义、测量方法、研究贡献、研究展望（与 V1 相同）
- 至少 4 类标签（与 V1 相同）
- Prompt 与 V1 相同

**笔记检测机制：**

```javascript
const hasNotebookTag = (item) => {
  const tags = item.getTags();
  return tags.some(tag => tag.tag && tag.tag.includes("📒"));
};

// 只有有📒标签的条目才处理
if (!hasNotebookTag(item)) {
  skipCount++;
  continue;
}
```

**内容获取优先级：**

```javascript
// 第一优先：笔记内容（BetterNotes Markdown）
const noteIds = item.getNotes();
if (noteIds.length > 0) {
  const noteItem = Zotero.Items.get(noteIds[0]);
  const noteContent = await Zotero.BetterNotes.api.convert.html2md(noteItem.getNote());
  if (noteContent && noteContent.trim().length > 50) {
    content = noteContent;
  }
}

// 第二优先：PDF前45000字符
if (!content) {
  const pdfItem = await item.getBestAttachment();
  if (pdfItem && pdfItem.isPDFAttachment()) {
    const fullTextData = await Zotero.PDFWorker.getFullText(pdfItem.id, 1);
    if (fullTextData && fullTextData.text && fullTextData.text.trim().length > 100) {
      content = fullTextData.text.substring(0, 45000).trim();
    }
  }
}

// 第三优先：摘要
if (!content) {
  const abstract = item.getField("abstractNote");
  if (abstract && abstract.trim().length > 50) {
    content = abstract.trim();
  }
}
```

**标签类别：**
- 与 V1 相同：4 类标签

**标签数量要求：**
- 与 V1 相同：至少 4 类标签

**保存方式：**
- 使用 `saveItemWithTags` 函数进行事务保存（与 V1 相同）

**与 V1-R 的区别：**

| 特性 | V1-R | V1-Rnote |
|------|------|----------|
| 删除旧标签 | ✅ 是 | ✅ 是（相同） |
| 内容获取策略 | PDF 前 45000 字符 → 摘要 | 笔记 → PDF 前 45000 字符 → 摘要（优先笔记） |
| 📒 标签要求 | ❌ 无 | ✅ 必须有 |
| 适用场景 | 所有条目 | 仅已阅读并做笔记的条目 |
| 保存方式 | ✅ saveItemWithTags | ✅ saveItemWithTags（相同） |

**与 V1 的区别：**

| 特性 | V1 | V1-Rnote |
|------|----|----------|
| 条目筛选 | 检查并跳过已有 4 类标签的条目 | 仅处理📒标签条目 |
| 删除旧标签 | ✅ 是（在跳过检查之后） | ✅ 是（与 R 系列相同） |
| 内容优先级 | PDF > 摘要 | 笔记 > PDF > 摘要 |
| 笔记支持 | ❌ | ✅ BetterNotes 转换 |
| 适用场景 | 所有条目 | 已阅读并做笔记的文献 |

**笔记内容优势：**
- 笔记是用户提炼和总结的内容
- 包含用户认为重要的要素信息
- 通常已经过人工筛选，要素识别更准确

**应用场景：**
- 已阅读并做笔记的文献
- 需要基于笔记内容识别条目细节
- 利用用户提炼的信息提高要素识别准确性

#### V2-Rnote - 基于 V2 的笔记优先版本（2025-10-29）
**命名说明：** `V2-Rnote` 表示这是**基于 V2 版本的笔记优先版本**。在 V2 的基础上增加了笔记优先逻辑（检查 📒 标签，优先获取笔记内容），同时会删除旧标签后重新生成（与 R 系列相同）。

**核心特性：**
- **删除现有 `V[1-5]-(def|mea|con|fut|sor)/` 标签后重新生成**（与 R 系列相同）

### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本系列
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，所有版本使用 PDF 前 45000 字符
4. **标签类别：** V1 系列为 4 类标签，V2 系列为 5 类标签（包含数据来源）
5. **数据来源识别：** V2 系列包含数据来源识别，需要注意区分数据来源与研究样本
6. **📒 标签：** Rnote 系列需要条目有 📒 标签才会处理
7. **重复处理：** R 系列会删除旧标签，请确认是否需要保留旧标签
8. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
9. **要素识别：** 要素识别准确性依赖于文献内容的质量和 AI 的理解能力
