## Tag6- 变量标签 版本迭代说明

### 概述
Tag6- 变量标签模块用于从学术文献中自动识别和提取研究变量标签，通过 AI 分析文献内容（摘要、PDF、笔记）识别 6 类关键变量：因变量（DV）、自变量（IV）、调节变量（MO）、中介变量（ME）、工具变量（INV）和控制变量（CV）。标签体系采用 `A1-DV/`, `A2-IV/`, `A3-MO/`, `A4-ME/`, `A5-INV/`, `A6-CV/` 前缀，便于分类管理和检索。


### 版本号体系说明
变量标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major].[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定内容获取策略、PDF 字符数、" 无 " 值处理逻辑和标签数量要求
  - V1-V2：主要版本迭代

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）
  - **-Rnote**：笔记优先版本（删除旧标签后重新生成，优先使用笔记内容，需要 📒 标签，基于对应的 V 版本）

**R 系列与 Rnote 系列的区别：**
- **相同点**：都会删除现有的 `A1-DV/` 到 `A6-CV/` 标签后重新生成
- **不同点**：
  - **R 系列**：内容获取策略与对应的 V 版本相同（如 V2-R 使用 PDF 前 30000 字符 → 摘要）
  - **Rnote 系列**：优先使用笔记内容，备选 PDF 或摘要（如 V1-Rnote 使用 笔记 → PDF 前 30000 字符 → 摘要）


#### 版本对应关系
| 基础版本 | 重新处理版本（R 系列） | 笔记优先版本（Rnote 系列） | 功能特性 |
|---------|---------------------|-------------------------|---------|
| V1 | V1-R | V1-Rnote | PDF 前 30000 字符优先，摘要备选，跳过所有 " 无 " 值变量，至少 2 个标签（DV 和 IV） |
| V2 | V2-R | V2-Rnote | PDF 前 30000 字符优先，摘要备选，只有 DV 为 " 无 " 时才添加，其他变量为 " 无 " 时跳过，至少 1 个标签（DV，即使是 DV/无） |

**命名逻辑说明：**
- **V1-R**：基于 V1 的重新处理版本（在 V1 基础上增加删除旧标签功能）
- **V2-R**：基于 V2 的重新处理版本（在 V2 基础上增加删除旧标签功能）
- **V1-Rnote**：基于 V1 的笔记优先版本（在 V1 基础上增加笔记优先逻辑）
- **V2-Rnote**：基于 V2 的笔记优先版本（在 V2 基础上增加笔记优先逻辑）

### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-10-13）
**核心特性：**
- 基础的变量标签生成功能
- 优先使用 PDF 前 30000 字符，备选摘要
- 跳过已处理条目机制
- 使用 `saveItemWithTags` 函数进行事务保存

**内容获取策略：**

```javascript
// 优先PDF前30000字符
const pdfItem = await item.getBestAttachment();
if (pdfItem && pdfItem.isPDFAttachment()) {
  const fullTextData = await Zotero.PDFWorker.getFullText(pdfItem.id, 1);
  if (fullTextData && fullTextData.text && fullTextData.text.trim().length > 100) {
    content = fullTextData.text.substring(0, 30000).trim();
  }
}

// 备选摘要
if (!content) {
  const abstract = item.getField("abstractNote");
  if (abstract && abstract.trim().length > 50) {
    content = abstract.trim();
  }
}
```

**标签格式：**
- `A1-DV/变量名`（因变量）
- `A2-IV/变量名`（自变量）
- `A3-MO/变量名`（调节变量）
- `A4-ME/变量名`（中介变量）
- `A5-INV/变量名`（工具变量）
- `A6-CV/变量名`（控制变量）

**" 无 " 值处理逻辑：**
- **跳过所有 " 无 " 值变量**（所有变量类型，包括 DV）
- 如果变量值为 " 无 "，则不添加该标签

**标签数量要求：**
- **至少 2 个标签**（DV 和 IV 必须同时存在）
- 如果缺少 DV 或 IV，标记为 " 格式错误 "

**AI Prompt 特点：**
- 详细说明 6 类变量的定义和识别要点
- 明确要求变量名称简洁准确（2-6 个字）
- 禁止使用描述性语句或完整句子作为变量名
- 严格区分自变量（IV）和控制变量（CV）
- 如果同一类别有多个变量，用加号（+）连接

**标签验证：**
- **AI 响应格式验证**：AI 返回的 JSON 必须包含 DV 和 IV 字段（即使值为 " 无 "），用于检查 AI 响应格式是否正确
- **" 无 " 值处理**：跳过所有 " 无 " 值变量（不添加标签）
- **最终标签数量要求**：至少要有 2 个标签（DV 和 IV）

**注意：** 格式验证（要求 AI 返回包含 DV 和 IV 字段的 JSON）与最终标签数量要求（至少 2 个标签）是两个不同的检查阶段。格式验证用于确保 AI 响应格式正确，而最终标签数量要求用于确保至少生成 2 个有效标签。

**保存方式：**
- 使用 `saveItemWithTags` 函数进行事务保存
- 在添加标签后统一保存，提高可靠性

**特点：**
- ✅ 基础功能完整
- ✅ 跳过机制，避免重复处理
- ✅ 使用事务保存，提高可靠性
- ✅ PDF 30000 字符，包含更多变量信息
- ❌ 所有 " 无 " 值都被跳过，即使 DV 为 " 无 " 也不标记

#### V2 - " 无 " 值处理优化（2025-10-13）
**核心改进：**
- 优化 " 无 " 值处理逻辑：只有 DV 为 " 无 " 时才添加标记，其他变量为 " 无 " 时跳过
- 标签数量要求从至少 2 个改为至少 1 个（DV，即使是 DV/无）
- 其他功能与 V1 相同

**" 无 " 值处理逻辑变化：**

```javascript
// V1: 跳过所有"无"值变量
if (value.trim() !== '' && value.trim() !== '无') {
  tagsToAdd.push(`${prefix}/${value.trim()}`);
}

// V2: 只有DV为"无"时才添加标记，其他变量为"无"时跳过
if (trimmedValue === '无') {
  if (key === 'DV') {
    tagsToAdd.push(`${prefix}/无`);
  }
  // 其他变量为"无"时不添加标签，直接跳过
} else {
  // 值不为"无"时,正常添加标签
  tagsToAdd.push(`${prefix}/${trimmedValue}`);
}
```

**标签数量要求变化：**

```javascript
// V1: 至少要有2个标签(DV和IV)
if (tagsToAdd.length < 2) {
  await saveItemWithTags(item, ['A1-DV/标签不完整'], 'fail');
  continue;
}

// V2: 至少要有1个标签(DV,即使是DV/无)
if (tagsToAdd.length < 1) {
  await saveItemWithTags(item, ['A1-DV/标签不完整'], 'fail');
  continue;
}
```

**标签验证：**
- **AI 响应格式验证**：AI 返回的 JSON 必须包含 DV 和 IV 字段（即使值为 " 无 "），用于检查 AI 响应格式是否正确
- **" 无 " 值处理**：只有 DV 为 " 无 " 时才添加标记，其他变量为 " 无 " 时跳过
- **最终标签数量要求**：至少要有 1 个标签（DV，即使是 DV/无）

**注意：** 格式验证（要求 AI 返回包含 DV 和 IV 字段的 JSON）与最终标签数量要求（至少 1 个标签）是两个不同的检查阶段。格式验证用于确保 AI 响应格式正确，而最终标签数量要求用于确保至少生成 1 个有效标签（通常是 DV）。

**原因说明：**
- 某些文献可能确实没有明确的 IV（例如纯描述性研究）
- DV 是研究结果的核心，即使为 " 无 " 也应该标记，便于识别问题文献
- 其他变量（IV、MO、ME、INV、CV）为 " 无 " 时通常表示文献中没有这些变量，不需要标记

**标签示例：**
- V1: 如果 DV=" 无 "，IV=" 数字化转型 "，则只添加 `A2-IV/数字化转型`（因为 DV 为 " 无 " 被跳过了）
- V2: 如果 DV=" 无 "，IV=" 数字化转型 "，则添加 `A1-DV/无` 和 `A2-IV/数字化转型`
- V2: 如果 DV=" 企业绩效 "，IV=" 无 "，则只添加 `A1-DV/企业绩效`（因为 IV 为 " 无 " 被跳过了）

**保存方式：**
- 与 V1 相同，使用 `saveItemWithTags` 函数进行事务保存

**Prompt：**
- 与 V1 相同

**变化对比：**

| 特性 | V1 | V2 |
|------|----|----|
| " 无 " 值处理 | 跳过所有 " 无 " 值变量 | 只有 DV 为 " 无 " 时才添加标记，其他变量为 " 无 " 时跳过 |
| 标签数量要求 | 至少 2 个标签（DV 和 IV） | 至少 1 个标签（DV，即使是 DV/无） |
| DV 为 " 无 " 的处理 | ❌ 不添加标签 | ✅ 添加 `A1-DV/无` 标签 |
| IV 为 " 无 " 的处理 | ❌ 不添加标签 | ❌ 不添加标签（跳过） |
| 适用场景 | 标准研究（必须有 DV 和 IV） | 更灵活，支持 DV 为 " 无 " 的情况 |

**后续版本基础：**
- V2 成为后续变体版本（V2-R）的基础版本

### R 系列（重新处理版本）迭代历史
**系列说明：**
- **R** = Re-process（重新处理）
- R 系列版本会：

```javascript
const removeExistingVariableTags = async (item) => {
  const existingTags = item.getTags();
  const variableTags = existingTags.filter(tag => 
    tag.tag && /^A[1-6]-(DV|IV|MO|ME|INV|CV)\//.test(tag.tag)
  );
  for (let variableTag of variableTags) {
    item.removeTag(variableTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingVariableTags(item);
```

### Rnote 系列（笔记优先版本）迭代历史
**系列说明：**
- **Rnote** = Re-process（重新处理）+ Note-priority（笔记优先）
- Rnote 系列版本会：
  1. **删除现有的 `A1-DV/` 到 `A6-CV/` 标签后重新生成**（与 R 系列相同）
  2. **检查 📒 标签**，仅处理带有 📒 标签的条目
  3. **优先使用笔记内容**，备选 PDF 或摘要
- **与 R 系列的区别**：
  - **相同点**：都会删除旧标签后重新生成
  - **不同点**：Rnote 系列优先使用笔记内容，而 R 系列使用与对应 V 版本相同的内容获取策略

#### V1-Rnote - 基于 V1 的笔记优先版本（2025-10-13）
**命名说明：** `V1-Rnote` 表示这是**基于 V1 版本的笔记优先版本**。在 V1 的基础上增加了笔记优先逻辑（检查 📒 标签，优先获取笔记内容），同时会删除旧标签后重新生成（与 R 系列相同）。

**核心特性：**
- **删除现有 `A1-DV/` 到 `A6-CV/` 标签后重新生成**（与 R 系列相同）
- **仅处理带有 📒 标签的条目**
- **优先使用笔记内容**（Zotero Notes，转换为 Markdown）
- 备选 PDF 前 30000 字符，然后摘要
- 跳过所有 " 无 " 值变量（与 V1 相同）
- 至少 2 个标签（DV 和 IV）（与 V1 相同）
- Prompt 与 V1 相同

**笔记检测机制：**

```javascript
const hasNotebookTag = (item) => {
  const tags = item.getTags();
  return tags.some(tag => tag.tag && tag.tag.includes("📒"));
};

// 只有有📒标签的条目才处理
if (!hasNotebookTag(item)) {
  skipCount++;
  continue;
}
```

**内容获取优先级：**

```javascript
// 第一优先：笔记内容（BetterNotes Markdown）
const noteIds = item.getNotes();
if (noteIds.length > 0) {
  const noteItem = Zotero.Items.get(noteIds[0]);
  const noteContent = await Zotero.BetterNotes.api.convert.html2md(noteItem.getNote());
  if (noteContent && noteContent.trim().length > 50) {
    content = noteContent;
  }
}

// 第二优先：PDF前30000字符
if (!content) {
  const pdfItem = await item.getBestAttachment();
  if (pdfItem && pdfItem.isPDFAttachment()) {
    const fullTextData = await Zotero.PDFWorker.getFullText(pdfItem.id, 1);
    if (fullTextData && fullTextData.text && fullTextData.text.trim().length > 100) {
      content = fullTextData.text.substring(0, 30000).trim();
    }
  }
}

// 第三优先：摘要
if (!content) {
  const abstract = item.getField("abstractNote");
  if (abstract && abstract.trim().length > 50) {
    content = abstract.trim();
  }
}
```

**" 无 " 值处理：**
- 跳过所有 " 无 " 值变量（与 V1 相同）

**标签数量要求：**
- 至少 2 个标签（DV 和 IV）（与 V1 相同）

**保存方式：**
- 使用 `item.saveTx()` 进行事务保存（与 V1-Rnote 和 V2-Rnote 相同，但实现方式与 V1/V2/V1-R/V2-R 不同）
- 直接在循环中添加标签，然后统一保存

**与 V1-R 的区别：**

| 特性 | V1-R | V1-Rnote |
|------|------|----------|
| 删除旧标签 | ✅ 是 | ✅ 是（相同） |
| 内容获取策略 | PDF 前 30000 字符 → 摘要 | 笔记 → PDF 前 30000 字符 → 摘要（优先笔记） |
| 📒 标签要求 | ❌ 无 | ✅ 必须有 |
| 适用场景 | 所有条目 | 仅已阅读并做笔记的条目 |

**与 V1 的区别：**

| 特性 | V1 | V1-Rnote |
|------|----|----------|
| 条目筛选 | 跳过已有标签 | 仅处理📒标签条目 |
| 删除旧标签 | ❌ 无 | ✅ 是（与 R 系列相同） |
| 内容优先级 | PDF > 摘要 | 笔记 > PDF > 摘要 |
| 笔记支持 | ❌ | ✅ BetterNotes 转换 |
| 适用场景 | 所有条目 | 已阅读并做笔记的文献 |

**笔记内容优势：**
- 笔记是用户提炼和总结的内容
- 包含用户认为重要的变量信息
- 通常已经过人工筛选，变量识别更准确

**应用场景：**
- 已阅读并做笔记的文献
- 需要基于笔记内容识别变量
- 利用用户提炼的信息提高变量识别准确性

### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本系列
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，V1/V2 使用 PDF 前 30000 字符
4. **" 无 " 值处理：** V1 跳过所有 " 无 " 值，V2 只有 DV 为 " 无 " 时才添加标记，其他变量为 " 无 " 时跳过

6. **📒 标签：** Rnote 系列需要条目有 📒 标签才会处理
7. **重复处理：** R 系列会删除旧标签，请确认是否需要保留旧标签
8. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
9. **变量识别：** 变量识别准确性依赖于文献内容的质量和 AI 的理解能力
