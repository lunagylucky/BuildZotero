## Tag5- 结论标签 版本迭代说明

### 概述
Tag5- 结论标签模块用于从学术文献中自动提取和生成研究结论标签，通过 AI 分析文献内容（摘要、PDF）识别主要研究发现、实证结果、理论贡献和实践启示。标签聚焦于研究结论和发现，避免与研究过程、方法、理论名称等标签重复。


### 版本号体系说明
结论标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major].[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定内容获取策略、PDF 字符数、标签格式和 Prompt 优化程度
  - V1-V2：主要版本迭代

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）


#### 版本对应关系
| 基础版本 | 重新处理版本（R 系列） | 功能特性 |
|---------|---------------------|---------|
| V1 | - | 摘要优先，PDF 前 3000 字符备选，无序号标签，固定 3 个标签 |
| V2 | V2-R | PDF 前 3000 字符优先，摘要备选，有序号标签，2-5 个标签（灵活），Prompt 优化 |

**命名逻辑说明：**
- **V2-R**：基于 V2 的重新处理版本（在 V2 基础上增加删除旧标签功能）

---


### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-08-08）
**核心特性：**
- 基础结论标签生成功能
- 优先使用摘要，备选 PDF 前 3000 字符
- 简单的 AI Prompt（中文）
- 跳过已处理条目机制

**内容获取策略：**

```javascript
// 优先摘要
const abstract = item.getField("abstractNote");
if (abstract && abstract.trim().length > 50) {
  content = abstract.trim();
} else {
  // 备选PDF前3000字符
  content = fullTextData.text.substring(0, 3000).trim();
}
```

**标签格式：**
- `Result/tag`（不带序号）
- 例如：`Result/正向影响`, `Result/调节效应`

**AI Prompt（中文）：**
- 生成 3 个结论标签
- 重点关注：主要研究发现和结论、实证结果和统计发现、理论贡献和实践启示、因果关系和影响效应、研究验证的假设结果
- 标签限制在 15 个中文字符以内

**特点：**
- ✅ 基础功能完整
- ✅ 跳过机制，避免重复处理
- ❌ 标签无序号，无法区分重要性
- ❌ 固定 3 个标签，不够灵活
- ❌ Prompt 未明确要求关系方向（正向、负向、U 型等）

---


#### V2 - PDF 优先策略与序号标签（2025-10-12）
**核心改进：**
- 调整内容获取优先级：优先 PDF，备选摘要
- 标签格式增加序号：`Result/1tag`, `Result/2tag`, ...
- 标签数量从固定 3 个改为灵活 2-5 个
- Prompt 优化，要求明确关系方向、调节效应、中介作用等

**内容获取策略变化：**

```javascript
// V1: 优先摘要
abstract > PDF前3000字符

// V2: 优先PDF
PDF前3000字符 > abstract
```

**原因说明：**
- PDF 内容更全面，包含摘要、引言、部分结果和结论
- 3000 字符包含更多结论相关信息
- 平衡信息量与处理效率

**标签格式变化：**

```javascript
// V1: 无序号
item.addTag('Result/' + tag);

// V2: 带序号
item.addTag('Result/' + (i + 1) + validTags[i]);
```

**标签示例：**
- V1: `Result/正向影响`, `Result/调节效应`, `Result/理论贡献`
- V2: `Result/1正向影响`, `Result/2调节效应`, `Result/3理论贡献`

**Prompt 优化：**
- 标签数量：2-5 个（根据文献复杂度）
- 字符限制：优先 15 个字符，可扩展到 20 个字符
- 明确要求：
  - 实证结果必须明确表述关系方向：正向、负向、倒 U 型、U 型等
  - 调节效应必须明确：正向调节、负向调节
  - 中介作用必须明确表述
- 按重要程度排序，第 1 个结论最重要
- 如果文献构建了理论模型、开发了测量工具、提出了新方法，应精准反映

**序号标签的优势：**
- 明确结论的重要性顺序
- 便于按重要性筛选和排序
- 支持优先使用最重要的结论

**变化对比：**

| 特性 | V1 | V2 |
|------|----|----|
| 内容优先级 | 摘要 > PDF 前 3000 | PDF 前 3000 > 摘要 |
| 标签格式 | `Result/tag` | `Result/1tag`, `Result/2tag`, ... |
| 标签数量 | 固定 3 个 | 2-5 个（灵活） |
| 字符限制 | 15 个字符 | 15-20 个字符（灵活） |
| 关系方向 | 未明确要求 | ✅ 明确要求（正向、负向、U 型等） |
| 调节/中介 | 未明确要求 | ✅ 明确要求（正向调节、负向调节、中介作用） |
| 重要性排序 | 未强调 | ✅ 强调按重要程度排序 |

**后续版本基础：**
- V2 成为后续变体版本（V2-R）的基础版本

### R 系列（重新处理版本）迭代历史
**系列说明：**
- **R** = Re-process（重新处理）
- R 系列版本会：
  1. **删除现有的 Result/ 标签后重新生成**
  2. 使用与对应 V 版本相同的内容获取策略（如 V2-R 使用 PDF 前 3000 字符 → 摘要）
  3. 处理所有条目（不需要 📒 标签）

---

#### V2-R - 基于 V2 的重新处理版本（2025-10-12）
**命名说明：** `V2-R` 表示这是**基于 V2 版本的重新处理版本**。在 V2 的基础上增加了删除旧标签后重新生成的功能。

**核心特性：**
- **删除现有 Result/ 标签后重新生成**（与 V 系列的区别）
- 优先 PDF 前 3000 字符，备选摘要（与 V2 相同）
- 标签格式：`Result/1tag`, `Result/2tag`, ...（带序号，与 V2 相同）
- 2-5 个标签（与 V2 相同）
- Prompt 与 V2 相同，要求明确关系方向、调节效应、中介作用等

**删除机制：**

```javascript
const removeExistingResultTags = async (item) => {
  const existingTags = item.getTags();
  const resultTags = existingTags.filter(tag => tag.tag && tag.tag.startsWith("Result/"));
  for (let resultTag of resultTags) {
    item.removeTag(resultTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingResultTags(item);
```

**内容获取策略：**
- 优先 PDF 前 3000 字符
- 备选摘要

**标签格式：**
- `Result/1tag`, `Result/2tag`, ...（带序号）

**Prompt：**
- 与 V2 相同，要求明确关系方向、调节效应、中介作用等

**与 V2 的区别：**

| 特性 | V2 | V2-R |
|------|----|------|
| 跳过机制 | ✅ 跳过已有标签的条目 | ❌ 不跳过，删除旧标签 |
| 删除机制 | ❌ 无 | ✅ 删除现有 Result/ 标签 |
| 适用场景 | 新增标签 | 更新/重新生成标签 |

**应用场景：**
- 需要更新已有结论标签
- Prompt 优化后重新生成标签
- 修正错误的标签
- V1 版本生成的标签需要升级为有序号格式

### 技术演进路径

#### 1. V1 → V2：功能优化
- 调整内容获取优先级（PDF 优先）
- 标签格式增加序号
- 标签数量从固定改为灵活（2-5 个）
- Prompt 优化，明确要求关系方向、调节效应、中介作用等
- 字符限制灵活化（15-20 个字符）


#### 2. V2 → V2-R：功能扩展
- 新增重新处理功能（删除旧标签）
- 适用于更新标签场景


### 使用建议

#### 选择版本指南

##### V 系列（标准版本）适用场景
- **V1：** 不推荐使用（功能不完善，无序号标签，固定 3 个标签）
- **V2：** **推荐** - 适合新增结论标签，需要序号标签，支持灵活标签数量（2-5 个），Prompt 优化


##### R 系列（重新处理版本）适用场景
- **V2-R：** **推荐** - 需要更新结论标签，删除旧标签后重新生成，基于 V2 的功能和 Prompt


#### 推荐版本总结
- **新增标签（标准场景）：** V2
- **更新标签（重新生成）：** V2-R


### 技术亮点

#### 1. 智能内容获取策略
- 多层级内容获取（摘要/PDF）
- 自动降级策略
- 字符长度限制，平衡质量与效率


#### 2. 序号标签系统
- 明确结论重要性顺序
- 便于筛选和排序
- 支持优先级使用


#### 3. 优化的 AI Prompt
- 中文 Prompt，更符合学术语境
- 明确要求关系方向（正向、负向、U 型等）
- 明确要求调节效应和中介作用
- 根据文献复杂度灵活调整标签数量（2-5 个）
- 避免与其他标签体系重复


#### 4. 场景化版本设计
- V 系列：标准新增标签
- R 系列：重新生成标签


#### 5. 完善的统计反馈
- 成功、跳过、无内容、失败、保存失败分类统计
- 便于监控处理效果和质量


### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本系列
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，建议使用 PDF 前 3000 字符
4. **标签格式：** V2/V2-R 系列使用序号标签，与 V1 版本不兼容
5. **重复处理：** R 系列会删除旧标签，请确认是否需要保留旧标签
6. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
7. **标签数量：** V2 及以后版本支持 2-5 个标签，根据文献复杂度自动调整
