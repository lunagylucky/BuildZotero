## Tag3- 样本标签 版本迭代说明

### 概述
Tag3- 样本标签模块用于从学术文献中自动提取和生成样本标签，通过 AI 分析文献内容（摘要、PDF、笔记）识别研究对象的三维特征。标签聚焦于研究层次（宏观/中观/微观）、研究领域/行业、具体样本/对象，帮助构建三维标签体系，避免与其他标签体系重复。


### 版本号体系说明
样本标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major]-[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定内容获取策略和 PDF 字符数
  - V1-V4：主要版本迭代

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）
  - **-Rnote**：笔记优先版本（删除旧标签后重新生成，优先使用笔记内容，需要 📒 标签，基于对应的 V 版本）

**R 系列与 Rnote 系列的区别：**
- **相同点**：都会删除现有的 `Sample/` 标签后重新生成
- **不同点**：
  - **R 系列**：内容获取策略与对应的 V 版本相同（如 V4-R 使用 PDF 前 25000 字符 → 摘要）
  - **Rnote 系列**：优先使用笔记内容，备选 PDF 或摘要（如 V4-Rnote 使用 笔记 → PDF 前 25000 字符 → 摘要）

**命名逻辑说明：**
- **V3-Rnote**：基于 V3 的笔记优先版本（在 V3 基础上增加笔记优先逻辑）
- **V4-Rnote**：基于 V4 的笔记优先版本（在 V4 基础上增加笔记优先逻辑）


#### 版本对应关系
| 基础版本 | 重新处理版本（R 系列） | 笔记优先版本（Rnote 系列） | 功能特性 |
|---------|---------------------|-------------------------|---------|
| V1 | - | - | 摘要优先，PDF 前 3000 字符，序号标签 |
| V2 | - | - | PDF 前 3000 字符优先，Prompt 优化 |
| V3 | V3-R | V3-Rnote | PDF 前 3000 字符优先，Prompt 更新（GPT 相关） |
| V4 | V4-R | V4-Rnote | PDF 前 25000 字符优先，大幅扩展 PDF 内容 |

### V 系列（标准版本）迭代历史

#### V1 - 初始版本（2025-07-31）
**核心特性：**
- 基础的样本标签生成功能
- 优先使用摘要，备选 PDF 前 3000 字符
- 简单的 AI Prompt（中文）
- 跳过已处理条目机制

**内容获取策略：**

```javascript
// 优先摘要（50字符阈值）
const abstract = item.getField("abstractNote");
if (abstract && abstract.trim().length > 50) {
  content = abstract.trim();
} else {
  // 备选PDF前3000字符
  content = fullTextData.text.substring(0, 3000).trim();
}
```

**标签前缀：**
- `Sample/` 前缀

**标签格式：**
- `Sample/1研究层次`, `Sample/2领域-研究领域`, `Sample/3样本-具体对象`
- 例如：`Sample/1中观-企业`, `Sample/2领域-创新管理`, `Sample/3样本-2000-2010年60家中国上市公司`

**标签数量：**
- 固定 3 个标签

**研究层次验证：**
- 10 种标准层次：
  - 宏观：国家、地区、行业、政策
  - 中观：企业、部门、网络
  - 微观：员工、高管、消费者

**无内容标记：**
- `Sample/无摘要`

**AI Prompt：**
- 基础的中文 Prompt，要求识别三个维度：
  1. 研究层次（必须从 10 种标准选择）
  2. 研究领域/行业（3-8 个字符）
  3. 具体样本/对象（限制在 15 个中文字符以内）

**特点：**
- ✅ 基础功能完整
- ✅ 跳过机制
- ✅ 序号标签系统
- ❌ Prompt 较简单，未提供研究领域示例
- ❌ 无内容标记不够通用（`Sample/无摘要`）

### R 系列（重新处理版本）迭代历史
**系列说明：**
- **R** = Re-process（重新处理）
- R 系列版本会：
  1. **删除现有的 `Sample/` 标签后重新生成**
  2. 使用与对应 V 版本相同的内容获取策略（如 V4-R 使用 PDF 前 25000 字符 → 摘要）
  3. 处理所有条目（不需要 📒 标签）

#### V3-R - 基于 V3 的重新处理版本（2025-10-12）
**版本说明：**
- 基于 V3 功能，增加删除旧标签后重新生成的能力
- 适用于需要更新或重新生成样本标签的场景

**核心特性：**
- 删除现有 `Sample/` 标签后重新生成
- 优先 PDF 前 3000 字符，备选摘要
- 标签格式：`Sample/1研究层次`, `Sample/2领域-研究领域`, `Sample/3样本-具体对象`（固定 3 个标签）
- Prompt 与 V3 相同

**删除机制：**

```javascript
const removeExistingSampleTags = async (item) => {
  const existingTags = item.getTags();
  const sampleTags = existingTags.filter(tag => tag.tag && tag.tag.startsWith("Sample/"));
  for (let sampleTag of sampleTags) {
    item.removeTag(sampleTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingSampleTags(item);
```

**与 V3 的区别：**

| 特性 | V3 | V3-R |
|------|----|------|
| 跳过机制 | ✅ 跳过已有标签的条目 | ❌ 不跳过，删除旧标签 |
| 删除机制 | ❌ 无 | ✅ 删除现有 Sample/ 标签 |
| 适用场景 | 新增标签 | 更新/重新生成标签 |

### Rnote 系列（笔记优先版本）迭代历史
**系列说明：**
- **Rnote** = Re-process（重新处理）+ Note-priority（笔记优先）
- Rnote 系列版本会：
  1. **删除现有的 `Sample/` 标签后重新生成**（与 R 系列相同）
  2. **检查 📒 标签**，仅处理带有 📒 标签的条目
  3. **优先使用笔记内容**，备选 PDF 或摘要
- **与 R 系列的区别**：
  - **相同点**：都会删除旧标签后重新生成
  - **不同点**：Rnote 系列优先使用笔记内容，而 R 系列使用与对应 V 版本相同的内容获取策略

---

#### V3-Rnote - 基于 V3 的笔记优先版本（2025-10-12）
**版本说明：**
- 基于 V3 功能，增加笔记内容优先处理能力
- 针对有 📒 标签的条目，优先使用笔记内容生成标签

**核心特性：**
- **删除现有 `Sample/` 标签后重新生成**（与 R 系列相同）
- 仅处理有 📒 标签的条目
- 内容获取优先级：笔记内容 > PDF 前 3000 字符 > 摘要
- 标签格式：`Sample/1研究层次`, `Sample/2领域-研究领域`, `Sample/3样本-具体对象`（固定 3 个标签）

**笔记检测机制：**

```javascript
const hasNotebookTag = (item) => {
  const tags = item.getTags();
  return tags.some(tag => tag.tag && tag.tag.includes("📒"));
};

// 只有有📒标签的条目才处理
if (!hasNotebookTag(item)) {
  skipCount++;
  continue;
}
```

**内容获取优先级：**

```javascript
// 第一优先：笔记内容（BetterNotes Markdown）
const noteContent = await Zotero.BetterNotes.api.convert.html2md(noteItem.getNote());

// 第二优先：PDF前3000字符
content = fullTextData.text.substring(0, 3000).trim();

// 第三优先：摘要
const abstract = item.getField("abstractNote");
```

**笔记内容优势：**
- 笔记是用户提炼和总结的内容
- 包含用户认为重要的样本信息
- 通常已经过人工筛选，质量更高

**与 V3-R 的区别：**

| 特性 | V3-R | V3-Rnote |

| 删除旧标签 | ✅ 是 | ✅ 是（相同） |

| 📒 标签要求 | ❌ 无 | ✅ 必须有 |
| 适用场景 | 所有条目 | 仅已阅读并做笔记的条目 |

**与 V3 的区别：**

| 特性 | V3 | V3-Rnote |
|------|----|----------|
| 条目筛选 | 跳过已有标签 | 仅处理📒标签条目 |

| 适用场景 | 所有条目 | 已阅读并做笔记的文献 |

### 使用建议

#### 选择版本指南

##### 标准版本（V 系列）适用场景
- **V1：** 不推荐使用（功能不完善，Prompt 较简单）
- **V2：** 适合 PDF 丰富的文献，需要详细的研究领域识别
- **V3：** **推荐** - 需要识别 GPT 相关研究，使用较短 PDF 内容，处理速度快
- **V4：** **推荐** - 需要完整样本识别，使用较长 PDF 内容，追求最佳准确性


##### 重新处理版本（V-R 系列）适用场景
- **V3-R：** 需要更新标签，使用较短 PDF 内容，处理速度快
- **V4-R：** **推荐** - 需要更新标签，追求更高质量，可接受较长处理时间


##### 笔记优先版本（V-Rnote 系列）适用场景
- **V3-Rnote：** 已做笔记的文献，使用较短 PDF 后备，处理速度快
- **V4-Rnote：** **推荐** - 已做笔记的文献，追求更高质量，可接受较长处理时间


#### 推荐版本总结
- **新增标签（标准场景）：** V4

- **基于笔记生成标签（快速）：** V3-Rnote

### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本和变体
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，V4/V4-R/V4-Rnote 处理时间较长（25000 字符）
4. **标签格式：** 所有版本使用统一的标签格式（`Sample/1层次`, `Sample/2领域-xxx`, `Sample/3样本-xxx`）
5. **📒 标签：** V-Rnote 系列需要条目有 📒 标签才会处理
6. **重复处理：** V-R 系列会删除旧标签，请确认是否需要保留旧标签
7. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
8. **笔记格式：** V-Rnote 系列需要 BetterNotes 插件支持
9. **研究层次：** 必须从 10 种标准层次中选择，不支持自定义
10. **样本格式：** 建议使用 " 年份 + 规模 + 具体样本名称 " 格式，例如 "2000-2010 年 60 家中国上市公司 "
