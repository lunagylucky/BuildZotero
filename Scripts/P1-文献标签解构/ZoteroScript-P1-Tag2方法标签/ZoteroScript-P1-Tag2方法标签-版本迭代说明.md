## Tag2- 方法标签 版本迭代说明

### 概述
Tag2- 方法标签模块用于从学术文献中自动提取和生成研究方法标签，通过 AI 分析文献内容（摘要、PDF、笔记）识别核心研究方法。标签体系包括：研究范式（第一标签）、核心分析方法（第二标签起），聚焦于方法学部分、模型构建、数据分析技术、统计方法和实验设计。


### 版本号体系说明
方法标签脚本采用统一的版本号体系，便于理解版本关系和功能特性：

#### 版本号格式：`V[major].[variant]`
- **V[major]（主版本号）**：表示核心功能版本，决定内容获取策略、PDF 字符数、标签前缀和 Prompt 优化程度
  - V1-V6：主要版本迭代

- **[variant]（变体标识）**：表示功能变体，在基础版本上增加特殊功能
  - 无标识：标准版本（跳过已有标签）
  - **-R**：重新处理版本（删除旧标签后重新生成，基于对应的 V 版本）
  - **-Rnote**：笔记优先版本（删除旧标签后重新生成，优先使用笔记内容，需要 📒 标签，基于对应的 V 版本）

**R 系列与 Rnote 系列的区别：**
- **相同点**：都会删除现有的 `sMeth/` 和 `Method/` 标签后重新生成
- **不同点**：
  - **R 系列**：内容获取策略与对应的 V 版本相同（如 V6-R 使用 PDF 前 30000 字符 → 摘要）
  - **Rnote 系列**：优先使用笔记内容，备选 PDF 或摘要（如 V6-Rnote 使用 笔记 → PDF 前 30000 字符 → 摘要）


#### 版本对应关系
| 基础版本 | 重新处理版本 | 笔记优先版本 | 功能特性 |
|---------|------------|------------|---------|
| V1 | - | V1-Rnote | Method/前缀，PDF 3000 字符，3 个固定标签 |
| V2 | - | - | Method/前缀，PDF 3000 字符，3 个固定标签，添加保存函数 |
| V3 | V3-R | V3-Rnote | sMeth/前缀，PDF 3000 字符，3 个固定标签（V3-Rnote 为 5 个） |
| V4 | V4-R | V4-Rnote | sMeth/前缀，PDF 30000 字符，3 个固定标签（V4-Rnote 为 5 个） |
| V5 | - | V5-Rnote | sMeth/前缀，PDF 30000 字符，2-5 个标签，Prompt 优化 |
| V6 | V6-R | V6-Rnote | sMeth/前缀，PDF 30000 字符，2-5 个标签，Prompt 进一步细化 |

```javascript
const saveItemWithTags = async (item, tags, countType) => {
  try {
    await Zotero.DB.executeTransaction(async function () {
      for (let tag of tags) {
        item.addTag(tag);
      }
      await item.save();
    });
    
    // 更新对应的计数器
    switch(countType) {
      case 'success': successCount++; break;
      case 'noContent': noContentCount++; break;
      case 'fail': failCount++; break;
    }
    
    return true;
  } catch (error) {
    console.error('保存标签失败:', error);
    errorCount++;
    return false;
  }
};
```

#### V3-R - 基于 V3 的重新处理版本（2025-10-12）
**版本说明：**
- 基于 V3 功能，增加删除旧标签后重新生成的能力
- 适用于需要更新或重新生成方法标签的场景

**核心特性：**
- 删除现有 `sMeth/` 和 `Method/` 标签后重新生成
- 优先 PDF 前 3000 字符，备选摘要
- 标签格式：`sMeth/1方法类型`, `sMeth/2方法`, `sMeth/3方法`（3 个固定标签）
- Prompt 与 V3 相同

**删除机制：**

```javascript
const removeExistingMethodTags = async (item) => {
  const existingTags = item.getTags();
  const methodTags = existingTags.filter(tag => 
    tag.tag && (tag.tag.startsWith("sMeth/") || tag.tag.startsWith("Method/"))
  );
  for (let methodTag of methodTags) {
    item.removeTag(methodTag.tag);
  }
};

// 在生成新标签前删除旧标签
await removeExistingMethodTags(item);
```

**与 V3 的区别：**

| 特性 | V3 | V3-R |
|------|----|------|
| 跳过机制 | ✅ 跳过已有标签的条目 | ❌ 不跳过，删除旧标签 |
| 删除机制 | ❌ 无 | ✅ 删除现有 sMeth/ 和 Method/ 标签 |
| 适用场景 | 新增标签 | 更新/重新生成标签 |

### 注意事项
1. **版本选择：** 根据实际需求选择合适的版本和变体
2. **内容质量：** 标签质量与输入内容质量相关，建议确保摘要或 PDF 质量
3. **处理时间：** PDF 内容长度会影响处理时间，V4/V5/V6 系列处理时间较长（30000 字符）
4. **标签格式：** V3 及以后版本使用 sMeth/ 前缀，与早期版本（Method/）不兼容

7. **重复处理：** V-R 系列会删除旧标签，请确认是否需要保留旧标签
8. **AI 依赖：** 所有版本依赖 AI 服务，需要稳定的网络连接
9. **笔记格式：** V-Rnote 系列需要 BetterNotes 插件支持
10. **方法识别准确性：** V6 系列使用最详细的 Prompt，方法识别准确性最高
11. **文件编辑错误：** V1 文件包含重复代码，建议使用后续版本
