# BuildZotero 简历项目介绍｜1 分钟版 & 3 分钟版

> **使用说明**：两版皆为口播稿。1 分钟版用于简历直通、开场自我介绍或「3 分钟内讲清楚你做了什么」类问题；3 分钟版用于专业面的项目讲述环节（产品/AI/数据/咨询/投行均适用）。全部数字口径与 [[项目-BuildZotero-01简历撰写]]、[[项目-BuildZotero-06项目介绍梳理]] 保持一致，不可临场新增事实。

---

## 一、一分钟版（≈ 220 字，口播 60 秒）

> 交付目标：在一分钟内让面试官**听到三件事** —— 做什么（WHAT）、为什么做（WHY）、做到了什么（HOW FAR）。

### 推荐稿（面向通用技术/产品岗）

「BuildZotero 是我独立设计与开发的 **AI 驱动学术知识工程解决方案**，定位是站在 Zotero + Obsidian + Cursor 开源生态之上，把非结构化论文转化为**可检、可改、可复用的结构化知识资产**。我围绕科研工作流解构出 **12 类瓶颈**，输出 PRD 与 7 模块 52 功能点的产品架构，独立实现 33 个功能的 JavaScript 脚本（累计 119 个版本文件），通过 **Prompt 工程 + 多模型路由（DeepSeek / Gemini 等）+ 二层检索**，把单篇文献结构化处理从 30 分钟压缩到 1 分钟以内，综述周期从 1–2 周压缩到 2–3 天，链路端到端耗时下降约 70%。产品已在**清华大学、中国科学技术大学**等多个实验室完成私有化部署与培训。我的核心价值不是『调了个大模型』，而是**做成了可复核、可追溯的知识工程**。」

### 岗位微调锚点（说完前 50 秒可加一句钩子）

- **产品经理岗**：「整条链路都有 PRD、功能清单、用户故事、架构文档和路线图做支撑。」
- **商业数据分析岗**：「全链路以 8 维标签 + 11 维分析协议的结构化 Schema 为底座，数据可做多维交叉透视。」
- **投行/咨询研究岗**：「这套『先治理—再检索—再生产』的方法论同构于**企业案例库、方法库**的建设。」
- **AI/AI Agent 岗**：「它是一个 7 模块异步代理框架，用二层检索优化 RAG 精度与上下文限制，建立了白盒化证据溯源链路。」

### 精简版（40 秒，电梯版）

「我做了一个叫 BuildZotero 的项目，面向科研人员，把 AI 和文献管理软件 Zotero 深度集成，解决『万级文献找不到、AI 写综述不可追溯、结论不可复核』三个痛点。我独立完成了需求调研、产品设计、脚本开发和私有化部署，目前服务清华、中科大等多个实验室。一句话定位：**做 AI 在科研场景的知识工程落地**。」

### 自检清单（上台前 30 秒扫）

- [ ] 说出**做什么**（AI 驱动学术知识工程）、**定位**（集成式方案，非独立 App）、**交付形态**（脚本 + Prompt + API + 文档）
- [ ] 说出**三个关键数字**：7 模块 / 52 功能点 / 119 版本文件；或 30 分钟→1 分钟、70% 提速、2 所高校
- [ ] 一句话**价值主张**：「可复核、可追溯、可复用的知识资产」，不说「调了个大模型」
- [ ] 数字口径与 [[项目-BuildZotero-02-任务主线与业务面试问答#数值、指标与材料口径速查|02 文数字速查]] 一致

## 二、三分钟版（≈ 750 字，口播 180 秒，STAR-R 展开）

> 交付目标：三分钟内呈现**问题→思考→方案→结果→反思**闭环；给面试官留 2–3 个可追问钩子。

### 脚本分段

**第 0 段｜10 秒｜定位一句话**

「BuildZotero 是我独立设计并开发的 **AI 驱动学术知识工程解决方案**，服务科研人员，目前在清华、中科大等多个实验室落地。它不是一个独立 App，而是站在 Zotero + Obsidian + Cursor 开源生态上的**集成式方案**，由自研 JavaScript 脚本 + Prompt 工程 + API 编排 + 15 份产品文档构成。」

**第 1 段｜40 秒｜Situation + Task（背景与目标）**

「起点是我自己的科研痛点：读过的论文找不到、写综述对不上来源、不同工具间手动搬运。我做了**结构化调研**，把痛点归纳到三个阶段、**12 类瓶颈**，最终抽象成 4 大技术壁垒——**上下文长度限制、AI 幻觉不可控、AI 输出不稳定、AI 与工作流断裂**。我的目标不是『做一个 ChatPDF 替代品』，而是把 AI 的一次性会话转化为**长期沉淀的知识资产**，让每一次 Token 消耗都能进到可检索、可复核的文库里。」

**第 2 段｜70 秒｜Action（做法核心三件事）**

「围绕这个目标我做了三件事：

**第一，产品架构**。把科研流程拆成 7 个模块——P0 基础操作、P1 文献标签解构、P2 标签统计、P3 自定义筛选、P4 综述、P5 引用、P6 交互系统，共 52 个功能点；设计 **8 维标签体系**（主题/方法/样本/理论/结论/变量/条目细节）和 **11 维分析框架**（研究背景、构念定义、测量方法、理论视角、对象层次等），把非结构化论文映射为标准化字段，直接写入 Zotero 可改、可版本、可检索。

**第二，代理编排**。搭了 7 模块异步代理框架支撑万级文库；设计**二层检索逻辑**——第一层是 AI 结构化字段做精准组合查询，第二层保留用户原标签体系不被破坏；同时建立**理论库循环更新机制**：AI 识别的新理论加 `Theory/new-` 前缀，人工核对后入库，形成闭环；最终所有回答都带 `[number]` 锚点，建立**白盒化证据溯源链路**。

**第三，集成部署**。基于成本与质量做**多模型路由**——复杂推理和长上下文用 Gemini 2.5/3.0，批量标准抽取用 DeepSeek V3/R1；通过 Zotero 插件打通 Obsidian 双向链接和 Cursor 终端，把整条链路耗时压了约 70%；最后完成清华、中科大等实验室的私有化部署和使用培训。」

**第 3 段｜40 秒｜Result（结果与量化）**

「结果上有三层：**单篇文献结构化处理从约 30 分钟压缩到 1 分钟以内**；**200+ 篇规模的综述任务引用来源 100% 可追溯**；**综述周期从 1–2 周压到 2–3 天**。工程层面交付了 33 个功能脚本、119 个版本迭代文件、15 份产品文档（PRD、功能清单、用户故事、架构、路线图、技术分析等）。落地层面被清华、中科大等多个实验室采纳并私有化部署。」

**第 4 段｜20 秒｜Reflection（反思与可迁移）**

「我从中得到的最大认知是：**AI 应用的核心壁垒不在模型调用，而在数据治理与人机协同**。AI 给草案，人对关键字段负责；这套**『先治理—再检索—再生产』**的方法论我认为可以同构地迁移到企业案例库、方法库、知识管理系统的建设上。」

### 可追问钩子（主动埋点）

| 钩子关键词 | 面试官大概率追问 | 展开路径 |
|---|---|---|
| **12 类瓶颈 / 4 大壁垒** | 你怎么做的用户调研？ | STAR：自身痛点→访谈→归类→竞品→优先级 |
| **7 模块 / 二层检索** | 为什么这么拆？ | 任务驱动拆模块，不按技术栈拆 |
| **多模型路由** | 具体怎么路由？ | 质量敏感 vs 成本敏感；任务分层 |
| **理论库循环** | 怎么防幻觉？ | 标准库 + `new` 标记 + 人验 + 白盒 |
| **私有化部署** | 数据安全怎么做？ | 本地优先 + 最小外发 + 课题组定制 |
| **70% 提速** | 数字怎么算的？ | 相对基线，分解到路由/批处理/去重解析 |

### 口播节奏建议

- 0–10 秒：定位（慢、清晰）
- 10–50 秒：背景与目标（稍快，但让 12/4 两个数字落耳）
- 50–120 秒：三件事（**最重要**，节奏要稳，每件事 20 秒）
- 120–160 秒：结果（重音落在三个具体数字）
- 160–180 秒：反思（收尾，留出气口给面试官接话）

## 三、岗位定制化开场（按岗位可替换「第 2 段」）

### 3.1 产品经理岗定制

第 2 段（70 秒）：「我做的第一件事是**需求结构化**——把科研群体 AI 应用需求系统化调研，按科研流程解构 12 类瓶颈，输出 PRD、功能清单、用户故事、产品架构、路线图和技术分析共 15 份产品文档；第二件事是**产品设计**——按用户真实工作流（入库→解构→检索→综述→引用→交互）而不是技术栈切出 7 个模块 52 个功能点，并用优先级矩阵排出 P0–P6；第三件事是**迭代验证**——走种子用户→口碑传播→私有化部署→使用培训的路径，累计 119 个版本迭代文件，每一轮迭代都锁定最痛的一类错来解决。」

### 3.2 商业数据分析岗定制

第 2 段（70 秒）：「我做的第一件事是**数据建模与 Schema 设计**——定义 8 维标签 + 11 维分析协议，把非结构化文本转成标准化字段入库；第二件事是**多维统计透视**——P2 模块做全景统计（条目/标签/自定义），P3 做多维逻辑筛选（变量 + 方法 + 样本组合），相当于一个垂直 BI；第三件事是**指标体系建立**——设计了标签抽取准确率 80–90%、理论库匹配 >95%、数据资产化率 >90% 的自测指标，并通过抽检与修订闭环持续优化。这套『先治理—再多维查询—再规模化产出』的链路，本质上和企业数据中台的治理→看板→自动化生产同构。」

### 3.3 投行/咨询研究岗定制

第 2 段（70 秒）：「我做的第一件事是**问题定义与市场扫描**——调研 Kimi、ChatPDF、NotebookLM、Elicit 等竞品的能力边界，明确差异化定位在『结构化 + 白盒化 + 工作流嵌入』；第二件事是**方法论设计**——建立 8 维 + 11 维分析框架，相当于对科研领域做了一套结构化的『案例库 Schema』，把非标知识转成可复核的结构化资产；第三件事是**交付与落地**——输出 15 份产品文档（PRD 到技术分析），并在清华、中科大等机构完成交付与培训。这套方法与**尽调工作底稿、咨询案例库、投研数据库**的建设思路高度同构——先定 Schema、再批量抽取、再多维检索与生产，最后沉淀为长期资产。」

### 3.4 AI / AI Agent 工程岗定制

第 2 段（70 秒）：「我做的第一件事是**架构设计**——7 模块异步代理框架支撑万级文库自动化，明确每个 Agent 的输入输出接口，形成分层深化的 Pipeline；第二件事是**RAG 工程化**——二层检索（结构化字段层 + 用户标签层）解耦，把 200–300 篇规模的输入做结构化组织而非塞进上下文，用带锚点的证据溯源降低幻觉，用理论库循环更新抑制 OOD 错误；第三件事是**模型与工程落地**——基于成本与质量做多模型路由（DeepSeek V3/R1 + Gemini 2.5/3.0），Prompt 三层设计（角色规范注入 + 任务格式约束 + 质量控制），并完成私有化部署。」

## 四、压力面补位（被打断怎么办）

### 场景 A：面试官 20 秒打断「别背简历，说人话」

快速切换到口语抓手：「一句话——**通用 AI 读论文像一次性便签，关掉会话就丢了；我做的是把知识装进可检索的柜子，标签进 Zotero 可以长期改、查、引用。**然后三件事……」

### 场景 B：面试官质疑数字「70% 怎么算的？」

立即回到口径：「相对优化前**同一条路径**的端到端耗时估计，分三块贡献：模型路由把简单任务甩到便宜模型省 30%；批处理减少重复解析 PDF 省 25%；缓存避免重复扣费省 15%。这是**设计目标与自测口径**，不是第三方审计数字。」

### 场景 C：面试官问「那你到底是产品还是工程师？」

「**产品主导型全栈个体**——产品设计（PRD、架构、路线图）是主战场，JavaScript 编码是把设计落地的手段。核心壁垒在 Prompt 与字段设计，不在炫语法。」
