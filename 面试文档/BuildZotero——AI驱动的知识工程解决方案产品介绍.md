# <span style="color:#3B8650;">产品概述</span>

> [!abstract] 产品概述
> <span style="color:#048562; font-weight:bold;">BuildZotero</span> 是基于 Zotero，专属于科研工作者的<span style="color:#048562; font-weight:bold;">AI 驱动的知识工程解决方案</span>。通过连通 <span style="color:#048562; font-weight:bold;">Zotero（文献管理与阅读）+ Obsidian（笔记看板与管理）+ Cursor 终端（笔记 Agent，直接通过 Cursor 管理笔记）</span>，构建完整的科研工作流闭环。<span style="color:#048562; font-weight:bold;">产品愿景</span>：<span style="color:#048562; font-weight:bold;">AI4S（AI for Science）</span>，专属于科研工作者的 AI Agent，融入科研工作者的工作流中，让 AI 真正地赋能科研工作。


## <span style="color:#43973D;">核心价值主张</span>

> [!info] 核心价值主张
> - <span style="color:#048562; font-weight:bold;">提升研究效率</span>：自动化文献标签提取与分类，节省 70%+ 的文献整理时间  
> - <span style="color:#048562; font-weight:bold;">AI 驱动分析</span>：基于大语言模型的智能内容理解，提取关键信息与理论框架  
> - <span style="color:#048562; font-weight:bold;">结构化输出</span>：生成标准化的文献综述矩阵与知识图谱，支持论文写作  
> - <span style="color:#048562; font-weight:bold;">工作流深度集成</span>：Zotero ↔ Obsidian ↔ Cursor 双向链接，打通科研工作流  
> - <span style="color:#048562; font-weight:bold;">知识资产化</span>：标签体系是永久的知识资产，可复用、可扩展  
> - <span style="color:#048562; font-weight:bold;">多维逻辑检索</span>：支持「变量 + 方法 + 样本」的交叉筛选，即使不记得文献名也能定位  
> - <span style="color:#048562; font-weight:bold;">理论库循环更新</span>：标准理论库匹配 + 新理论标记，理论库持续增值  
> - <span style="color:#048562; font-weight:bold;">模型配置策略</span>：不同任务配置不同的大模型以节省 Tokens 费用

## <span style="color:#43973D;">核心范式转移</span>

> [!info] 核心范式转移
> BuildZotero 与传统 AI 阅读工具的根本区别见下表。

| <span style="color:#048562; font-weight:bold;">维度</span> | <span style="color:#048562; font-weight:bold;">传统 AI 阅读工具</span> | <span style="color:#048562; font-weight:bold;">BuildZotero</span> |
|---------|-------------------|----------------|
| <span style="color:#048562; font-weight:bold;">产品定位</span> | 通用 AI 工具 | 专属于科研工作者的 AI Agent |
| <span style="color:#048562; font-weight:bold;">交互逻辑</span> | 点状会话，知识随 Session 结束而损耗 | 结构化索引，AI 提取转化为永久资产 |
| <span style="color:#048562; font-weight:bold;">检索能力</span> | 关键词检索，必须记得文献名 | 多维逻辑检索，支持标签组合查询 |
| <span style="color:#048562; font-weight:bold;">处理规模</span> | 小批量，受限于 Context Window | 超大规模，支持数千篇文献统一打标 |
| <span style="color:#048562; font-weight:bold;">可维护性</span> | 黑盒不可改，AI 总结错了难修正 | 白盒化维护，标签 100% 可修改 |
| <span style="color:#048562; font-weight:bold;">工作流集成</span> | 独立工具，形成信息孤岛 | Zotero ↔ Obsidian ↔ Cursor 双向链接 |
| <span style="color:#048562; font-weight:bold;">理论库管理</span> | 无理论库，无法验证理论标签 | 理论库循环更新，持续增值 |
| <span style="color:#048562; font-weight:bold;">模型配置</span> | 单一模型 | 多模型配置策略（节省 Tokens） |

# <span style="color:#3B8650;">产品架构</span>

> [!info] 架构总览
> <span style="color:#048562; font-weight:bold;">核心架构</span>：系统管理（P0）+ 文献标签系统（P1）+ 全局查询与统计（P2）+ 结构化检索与自定义检索标准（P3）+ 高质量文献综述生成（P4）+ 文献引用（P5）+ 覆盖各场景的交互（P6）

## <span style="color:#43973D;">模块化设计理念</span>

系统采用模块化架构，每个模块独立运行，可根据研究需求灵活组合使用：

```
BuildZotero
├── P0: 基础操作模块        # 标签管理、标题清理等基础功能
├── P1: 文献标签解构模块    # 多维度标签提取（主题、方法、理论等）- 核心
├── P2: 文献标签统计模块    # 全景情报看板与统计透视
├── P3: 文献自定义筛选模块  # 基于标签的智能筛选 - 多维逻辑检索
├── P4: 文献综述模块        # 自动化文献综述生成 - 结构化矩阵
├── P5: 文献引用模块        # 智能引用添加
└── P6: 交互系统模块        # 证据溯源型对话 + 工作流集成
```

---

### <span style="color:#7CA531;">多维标注体系驱动的技术架构</span>

BuildZotero 采用 <span style="color:#048562; font-weight:bold;">多维标注体系驱动的 Prompt 工程架构</span>，通过精心设计的学术标注规范（8 维标签体系 + 11 维分析框架）实现智能化的文献内容提取。核心贡献是解决了 " 数据质量的确定性 " 问题：通过标准化的标注规范，强制让 AI 将非结构化文献转化为高质量的结构化数据。

<span style="color:#048562; font-weight:bold;">技术流程</span>：

```
非结构化PDF → AI 语义解构器 → 结构化标签云 → Zotero 关系型数据库
                                                      ↓
                    多维组合检索 / 自动化矩阵生成 / 零幻觉对话
```

### <span style="color:#7CA531;">工作流集成架构</span>

<span style="color:#048562; font-weight:bold;">Zotero ↔ Obsidian ↔ Cursor 双向链接</span>：

- ✅ <span style="color:#048562; font-weight:bold;">Zotero</span>：作为自己的文献库以及图书馆
- ✅ <span style="color:#048562; font-weight:bold;">Obsidian</span>：笔记看板以及笔记整理
- ✅ <span style="color:#048562; font-weight:bold;">Cursor</span>：笔记 Agent 管理以及操作
- ✅ <span style="color:#048562; font-weight:bold;">P6 交互笔记</span>：与 Obsidian 双向链接，同步更新
- ✅ <span style="color:#048562; font-weight:bold;">打通科研工作流</span>：让 AI 真正地赋能科研工作

## <span style="color:#43973D;">🎨 核心功能模块</span>

### <span style="color:#7CA531;">P0 - 基础操作模块</span>

<span style="color:#048562; font-weight:bold;">定位</span>：系统基础能力层，提供文献管理的通用工具

<span style="color:#048562; font-weight:bold;">核心功能</span>：
- <span style="color:#048562; font-weight:bold;">标签展示</span>：自定义标签属性列的需要展示的标签，可根据当前的需求自定义，比如，当前学者正在做方法论上的研究，那么就可以只显示方法标签，如果正在做专题研究，则可以显示主题、变量等标签，所有的标签高度自定义，按照正则表达式，随意组合和展示
- <span style="color:#048562; font-weight:bold;">标签清理</span>：智能清理重复或无效标签
- <span style="color:#048562; font-weight:bold;">标签删除</span>：批量删除指定标签
- <span style="color:#048562; font-weight:bold;">标题清理</span>：规范化文献标题格式

<span style="color:#048562; font-weight:bold;">适用场景</span>：
- 初次使用系统的数据清洗
- 定期维护文献库质量
- 批量规范化操作

---

### <span style="color:#7CA531;">P1 - 文献标签解构模块（核心）</span>

<span style="color:#048562; font-weight:bold;">定位</span>：核心能力层，通过 AI 自动提取文献的多维信息标签

<span style="color:#048562; font-weight:bold;">核心目标</span>：利用 Prompt 工程将 PDF 全文强制映射为一套标准化的学术标注规范（逻辑结构协议）

<span style="color:#048562; font-weight:bold;">多维标注体系与标准化治理</span>:
- <span style="color:#048562; font-weight:bold;">分析规范设计</span>：独立设计一套包含 11 个核心维度的学术分析协议（Standardized Protocol），涵盖核心变量（IV/DV）、理论架构、研究方法与样本特征等
- <span style="color:#048562; font-weight:bold;">数据结构化映射</span>：利用提示词工程驱动 LLM 将长文本核心逻辑映射为 Zotero 的标准化标签体系，实现了从 " 非结构化文献 " 向 " 标准化知识组件 " 的物理级转换

<span style="color:#048562; font-weight:bold;">核心功能</span>：

| 标签类型              | 功能说明        | 输出格式                          | 应用场景          | 标注规范设计                      |
| ----------------- | ----------- | ----------------------------- | ------------- | --------------------------- |
| **Tag1 - 主题标签**   | 提取研究主题与核心概念 | `Item/序号`                     | 主题聚类、研究方向识别   | 关键变量、核心概念                   |
| **Tag 2- 方法标签**   | 识别研究方法与技术   | `sMeth/1-6`                   | 方法比较、研究设计分析   | 方法论指纹：研究类型、主要分析、辅助分析        |
| **Tag 3 - 样本标签**  | 提取研究层次与样本特征 | `Sample/1-3`                  | 样本特征统计、研究范围分析 | 研究层次、研究领域、研究样本              |
| **Tag 4 - 理论标签**  | 提取理论框架与机制   | `Theory/序号` 或 `Theory/new-序号` | 理论梳理、机制分析     | **标准理论库匹配 + 新理论标记**         |
| **Tag 5- 结论标签**   | 提取研究发现与结论   | `Result/序号`                   | 结果对比、发现汇总     | 主要结论、次要结论                   |
| **Tag 6- 变量标签**   | 提取变量定义与关系   | `A1-DV/`, `A2-IV/` 等          | 变量关系分析、模型构建   | **因果链条**：因变量、自变量、中介变量、调节变量等 |
| **Tag 7- 条目细节标签** | 提取研究贡献与展望   | `V1-def/`, `V3-con/` 等        | 研究贡献分析、未来方向识别 | 变量定义、测量方法、研究贡献、未来展望         |

<span style="color:#048562; font-weight:bold;">版本体系</span>：
- <span style="color:#048562; font-weight:bold;">V 系列</span>：标准版本，基于 PDF 或摘要的内容提取
- <span style="color:#048562; font-weight:bold;">V-R 系列</span>：重新处理版本，支持标签更新与删除
- <span style="color:#048562; font-weight:bold;">V-Rnote 系列</span>：笔记优先版本，优先使用用户笔记内容

<span style="color:#048562; font-weight:bold;">理论库构建与循环更新机制</span>:
- ✅ <span style="color:#048562; font-weight:bold;">标准理论库匹配</span>：理论标签基于标准理论库进行标准匹配
- ✅ **新理论标记**：AI 从文献中总结出的理论标签将会加上 `new` 标记
- ✅ <span style="color:#048562; font-weight:bold;">用户核对</span>：用户可以去根据文献内容去核对
- ✅ <span style="color:#048562; font-weight:bold;">理论库更新</span>：如果这是一项标准理论库没有的，那么则可以纳入标准理论库
- ✅ <span style="color:#048562; font-weight:bold;">循环更新</span>：循环下去，自己的标准理论库就会一直更新与完善
- ✅ <span style="color:#048562; font-weight:bold;">标签正确性</span>：确保了所有理论标签的正确性，解决了理论查找困难的问题
- ✅ <span style="color:#048562; font-weight:bold;">知识资产</span>：这些理论库是自己的知识资产

<span style="color:#048562; font-weight:bold;">资产维护特性</span>：
- ✅ 生成的标签直接写入 Zotero 数据库
- ✅ <span style="color:#048562; font-weight:bold;">用户可在阅读过程中随时手动修改标签，确保数据库的 100% 准确性</span>
- ✅ 支持增量更新，避免重复处理

### <span style="color:#7CA531;">P2 - 文献标签统计模块（全景情报看板与统计透视）</span>

<span style="color:#048562; font-weight:bold;">定位</span>：数据分析层，提供标签统计与可视化能力

<span style="color:#048562; font-weight:bold;">全景情报看板与统计透视</span>:
- <span style="color:#048562; font-weight:bold;">全局态势感知（Situational Awareness）</span>：开发 P2 统计模块，打破了 " 单点检索 " 的局限。通过将文献库情报表格化、看板化呈现，实现了从 " 条目逻辑 " 到 " 标签逻辑 " 的全量透视
- <span style="color:#048562; font-weight:bold;">动态回溯与穿透</span>：支持对研究议题分布、方法论占比等核心指标进行统计。所有统计结果均支持秒级回溯（Traceability），确保研究者在掌握宏观趋势的同时，能瞬间下钻定位至具体的文献来源

<span style="color:#048562; font-weight:bold;">核心功能</span>：
- <span style="color:#048562; font-weight:bold;">Table1 - 统计条目</span>：生成文献条目统计表格
- <span style="color:#048562; font-weight:bold;">Table2 - 统计标签</span>：生成标签使用频率统计

<span style="color:#048562; font-weight:bold;">适用场景</span>：
- 研究领域概览
- 标签使用趋势分析
- 文献库质量评估
- 全局态势感知

### <span style="color:#7CA531;">P3 - 文献自定义筛选模块（多维逻辑检索）</span>

<span style="color:#048562; font-weight:bold;">定位</span>：智能筛选层，基于标签规则进行精准筛选

<span style="color:#048562; font-weight:bold;">核心目标</span>：利用打标后的结构化数据，实现 " 逻辑组合寻找文献 "

<span style="color:#048562; font-weight:bold;">原生功能赋能与二层检索架构</span>:
- <span style="color:#048562; font-weight:bold;">激活原生搜索价值</span>：通过注入结构化标签，极大程度发挥了 Zotero 高级检索（Advanced Search）的实战价值，将原本单一的关键词搜索升级为 " 变量 + 方法 + 样本 " 的复合逻辑筛选
- <span style="color:#048562; font-weight:bold;">二层检索逻辑（Two-tier Retrieval）</span>：支持流程化、高度自定义的二次检索与标注。该设计确保了 AI 结构化标签与用户原有标签架构的解耦（Decoupling），用户可完全自主地进行标签清理与删除管理，不干扰原有的分类秩序

<span style="color:#048562; font-weight:bold;">典型场景</span>：
> " 帮我找出所有：<span style="color:#048562; font-weight:bold;">[变量包含：数字化]</span> 且 <span style="color:#048562; font-weight:bold;">[方法包含：双重差分]</span> 且 <span style="color:#048562; font-weight:bold;">[样本包含：制造业]</span> 的文献。"

<span style="color:#048562; font-weight:bold;">核心功能</span>：

| 筛选类型 | 功能说明 | 筛选规则 |
|---------|---------|---------|
| **TagM1 - 变量定义与衡量** | 筛选特定变量的定义与测量方法 | 基于 `V1-def/`, `V2-mea/` 标签 |
| <span style="color:#048562; font-weight:bold;">TagM2 - 主题变量分类</span> | 筛选特定主题的变量研究 | 基于变量标签 + 关键词匹配 |
| **TagM3 - 控制变量类型** | 筛选特定类型的控制变量 | 基于 `A6-CV/` 标签 |
| <span style="color:#048562; font-weight:bold;">TagM4 - 地区固定效应</span> | 筛选固定效应相关研究 | 基于地区、年份、行业关键词 |

<span style="color:#048562; font-weight:bold;">价值点</span>：彻底解决 " 我记得读过这类文章，但死活找不到 " 的科研痛点。

### <span style="color:#7CA531;">P4 - 文献综述模块（结构化综述矩阵生成）</span>

<span style="color:#048562; font-weight:bold;">定位</span>：内容生成层，自动化生成结构化文献综述

<span style="color:#048562; font-weight:bold;">核心目标</span>：将 " 数据库数据 " 转化为 " 学术产出文稿 "，不再是基于摘要乱写，而是读取已校准的 P1 标签，自动填充 11 维综述矩阵

<span style="color:#048562; font-weight:bold;">逻辑驱动的智能生产</span>:
- <span style="color:#048562; font-weight:bold;">高质量生成任务</span>：利用 " 预先治理、逻辑对齐 " 的输入策略，显著提升了 RAG 在文献综述、文档问答场景下的严谨性。由于输入端是高度逻辑化的 " 知识组件 "，生成内容的逻辑深度远超常规 AI 阅读工具
- <span style="color:#048562; font-weight:bold;">全链路证据溯源</span>：确保所有生成任务均具备物理级溯源能力，支持一键回溯来源，从根源抑制 AI 幻觉，确保科研产出的 100% 确定性
- <span style="color:#048562; font-weight:bold;">大规模输入能力</span>：本系统一次性同时将两三百篇文献作为输入，这是目前的 AI 阅读工具以及其他的 AI 科研工具无法达到的

<span style="color:#048562; font-weight:bold;">核心功能</span>：

#### <span style="color:#B6AB44;">LR0 - 摘要简洁</span>

- 清理摘要中的翻译标记，保留原始内容

#### <span style="color:#B6AB44;">LR1 - 矩阵标签生成</span>

生成 <span style="color:#048562; font-weight:bold;">11 维文献综述矩阵</span>，支持两种输出格式：

| 版本 | 输出格式 | 列数 | 引用格式 | 适用场景 |
|------|---------|------|---------|---------|
| <span style="color:#048562; font-weight:bold;">V1-5AY</span> | 5 列表格 | 5 列 | （作者，年份，期刊缩写) 编号 | 详细分析，包含要点、发现、脉络、段落、核心文献 |
| <span style="color:#048562; font-weight:bold;">V1-2AYJ</span> | 2 列表格 | 2 列 | （作者，年份，期刊缩写) 编号 | 简洁输出，推荐使用，星级优先排序 |
| <span style="color:#048562; font-weight:bold;">V1-2YJ</span> | 2 列表格 | 2 列 | （年份，期刊缩写) 编号 | 简洁输出，不需要作者信息 |

<span style="color:#048562; font-weight:bold;">分析维度（11 维）</span>：
1. 研究背景 2. 构念定义 3. 测量方法 4. 理论视角 5. 对象层次  
6. 研究情境 7. 研究方法 8. 前因因素 9. 作用后果 10. 中介机制 11. 调节因素

#### <span style="color:#B6AB44;">LR2 - 矩阵标签要点</span>

生成要点式文献综述，支持 3-7 列表格格式

#### <span style="color:#B6AB44;">LR3 - 矩阵标签测量</span>

生成变量定义与衡量的专门综述表格（6 列）

#### <span style="color:#B6AB44;">LR4 - 文献综述（研究议程识别）</span>

生成研究议程矩阵（4 列），<span style="color:#048562; font-weight:bold;">基于标签分布，自动识别该领域的 3-5 个核心研究议程（Research Agendas）</span>

## <span style="color:#43973D;">💡 使用场景示例</span>

### <span style="color:#7CA531;">场景 1：在海量文献中查找所需文献</span>

<span style="color:#048562; font-weight:bold;">需求</span>：找出所有研究 " 数字化转型 " 且使用 " 双重差分 " 方法且样本为 " 制造业企业 " 的文献

<span style="color:#048562; font-weight:bold;">操作流程</span>：
1. 使用 P1 模块提取所有相关文献的标签
